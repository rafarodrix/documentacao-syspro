## Objetivo

Este script automatiza a criação de novos registros de estoque (`produto_qtde`) baseando-se em um grupo de estoque já existente. É ideal para cenários onde uma nova unidade ou depósito foi criado e precisa herdar as configurações e quantidades de um depósito base.

> **Importante:** O script gera novos UUIDs para cada registro inserido, garantindo a integridade da chave primária. **Realize backup antes da execução.**

## Script SQL (Firebird 5.0)

```sql
-- Script: Duplicação de registros de estoque para um novo grupo
-- Parâmetros: 
-- :ID_GRUPO_ESTOQUE_ORIGEM  -> UUID do grupo base
-- :ID_GRUPO_ESTOQUE_DESTINO -> UUID do novo grupo

INSERT INTO produto_qtde (
    id_produto_qtde,
    id_produto,
    id_empresa,
    id_grupo_estoque,
    id_lote_ativo,
    ativo,
    qtde,
    qtde_min,
    qtde_max,
    dtc_saida_ultima,
    dtc_entrada_ultima,
    informativo,
    localizacao,
    dt_alteracao,
    hr_alteracao,
    id_usuario
)
SELECT
    '{' || UUID_TO_CHAR(GEN_UUID()) || '}',
    pq.id_produto,
    pq.id_empresa,
    :ID_GRUPO_ESTOQUE_DESTINO,
    pq.id_lote_ativo,
    pq.ativo,
    pq.qtde,
    pq.qtde_min,
    pq.qtde_max,
    pq.dtc_saida_ultima,
    pq.dtc_entrada_ultima,
    pq.informativo,
    pq.localizacao,
    CURRENT_DATE, -- Atualizado para data atual
    CURRENT_TIME, -- Atualizado para hora atual
    pq.id_usuario
FROM
    produto_qtde pq
WHERE
    pq.id_grupo_estoque = :ID_GRUPO_ESTOQUE_ORIGEM;

COMMIT;
```
