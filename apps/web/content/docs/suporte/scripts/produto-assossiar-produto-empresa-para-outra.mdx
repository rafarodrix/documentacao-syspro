## Objetivo

Este script permite replicar o catálogo de produtos associados a uma empresa (origem) para uma nova empresa (destino). É ideal para processos de expansão, onde uma nova filial precisa herdar as mesmas definições de produtos comerciais da matriz ou de outra unidade já operacional.

> **Atenção:** O script gera novos UUIDs para cada vínculo na tabela `produto_comercial_empresa`. **Backup obrigatório antes da execução.**

## Script SQL (Firebird 5.0)

```sql
-- Script: Replicar associação de produtos entre empresas
-- Parâmetros: 
-- :ID_EMPRESA_ORIGEM  -> UUID da empresa base
-- :ID_EMPRESA_DESTINO -> UUID da nova empresa que receberá o catálogo

INSERT INTO produto_comercial_empresa (
    id_produto_comercial_empresa,
    id_produto_comercial,
    id_empresa,
    ativo,
    dt_alteracao,
    hr_alteracao
)
SELECT
    '{' || UUID_TO_CHAR(GEN_UUID()) || '}',
    pce.id_produto_comercial,
    :ID_EMPRESA_DESTINO,
    pce.ativo,
    CURRENT_DATE,
    CURRENT_TIME
FROM
    produto_comercial_empresa pce
WHERE
    pce.id_empresa = :ID_EMPRESA_ORIGEM;

COMMIT;
```
