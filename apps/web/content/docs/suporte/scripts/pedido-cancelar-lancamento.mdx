## Objetivo

Este script realiza a limpeza da base de dados, invalidando pedidos que ficaram "presos" no status de edição (`LANCANDO`) e que possuem numeração zerada. Isso geralmente ocorre por quedas de conexão ou abandono da venda pelo usuário, o que pode poluir relatórios e consultas.

> **Atenção:** É obrigatório realizar um backup completo da base de dados antes da execução. Valide o impacto em um ambiente de homologação primeiro.

## Script SQL (Firebird 5.0)

```sql
-- Script: Cancelamento de Pedidos Antigos em Lançamento
-- Objetivo: Inativar e cancelar pedidos não finalizados (número 0, status 'LANCANDO').
-- Recomendação: Ajuste a data de corte conforme a necessidade do cliente.

UPDATE pedido AS ped
SET ped.status = 'CANCELADO',
    ped.ativo = '0'
WHERE ped.numero = 0
  AND ped.ativo = 1
  AND ped.status = 'LANCANDO'
  AND ped.dt_venda < '2025-10-16'; -- <<< INFORME AQUI A DATA DE CORTE (YYYY-MM-DD)

COMMIT;
```
