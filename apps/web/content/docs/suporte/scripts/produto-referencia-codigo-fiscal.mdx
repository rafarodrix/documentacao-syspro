## Objetivo

Cria um registro na tabela `produto_referencia` para todos os produtos, associando-os a um participante específico (ex: matriz) e utilizando o próprio código fiscal do produto como código de referência. 

Este procedimento é essencial para operações de transferência entre matriz e filial, garantindo que o sistema reconheça o item pelo código fiscal durante a importação de XMLs.

> **ATENÇíO:** O script não duplica referências caso elas já existam para o participante informado. **Backup obrigatório antes da execução.**

## Script SQL (Firebird 5.0)

```sql
-- Script: Adicionar codigo fiscal como referencia de produto para um participante
-- Parâmetros: :ID_PARTICIPANTE_FORNECEDOR (UUID do participante matriz/fornecedor)

INSERT INTO produto_referencia (
    id_produto_referencia,
    id_produto,
    id_participante_for,
    ativo,
    cd_produto_fornecedor,
    dt_ref,
    hr_ref,
    dt_alteracao,
    hr_alteracao
)
SELECT
    '{' || UUID_TO_CHAR(GEN_UUID()) || '}',
    pr.id_produto,
    :ID_PARTICIPANTE_FORNECEDOR,
    1,
    pr.cd_fiscal,
    CURRENT_DATE,
    CURRENT_TIME,
    CURRENT_DATE,
    CURRENT_TIME
FROM
    produto pr
WHERE
    NOT EXISTS (
        SELECT 1
        FROM produto_referencia pr_ex
        WHERE pr_ex.id_produto = pr.id_produto
          AND pr_ex.id_participante_for = :ID_PARTICIPANTE_FORNECEDOR
    );

COMMIT;
```
