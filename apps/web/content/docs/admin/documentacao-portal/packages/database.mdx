# Package: @dosc-syspro/database

> Prisma ORM — schema PostgreSQL, client gerado e seed.
> Atualizado em: 2026-05-05

---

## Responsabilidade

`@dosc-syspro/database` centraliza:
- Schema Prisma (fonte de verdade do banco)
- PrismaClient gerado e exportado
- Tipos TypeScript gerados pelo Prisma
- Seed de dados iniciais

Não contém lógica de negócio — apenas ORM e tipos.

---

## Exports

```typescript
// Re-exporta tudo do Prisma gerado
export { PrismaClient } from '@prisma/client'
export type { Company, RemoteHost, RemoteSession, ... } from '@prisma/client'
```

## Configuração de conexão

```bash
# Pool via PgBouncer (para a aplicação)
DATABASE_URL=postgresql://user:pass@pgbouncer:5432/trilink?pgbouncer=true

# Conexão direta (para migrations e seed)
DIRECT_URL=postgresql://user:pass@postgres:5432/trilink
```

O Prisma usa `DATABASE_URL` para queries e `DIRECT_URL` para migrations (evita problemas com pool).

## Binary targets

Configurados no `schema.prisma` para suportar múltiplos ambientes:

```prisma
generator client {
  provider      = "prisma-client-js"
  binaryTargets = ["native", "linux-musl-openssl-3.0.x"]
}
```

- `native`: desenvolvimento local (macOS, Windows, Linux glibc)
- `linux-musl-openssl-3.0.x`: container Docker Alpine

## Seed

```bash
cd packages/database
npx prisma db seed
```

O seed (`prisma/seed.ts`) cria dados iniciais de desenvolvimento:
- Empresa demo
- Usuário admin
- Configurações padrão
