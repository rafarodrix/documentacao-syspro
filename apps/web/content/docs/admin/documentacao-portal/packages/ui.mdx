## Sistema de Tokens

Arquivo: `packages/ui/src/tokens.css` — **fonte única de verdade** para design tokens.

### Como os tokens chegam ao app

```
tokens.css
  └── re-exportado por packages/ui (export "./tokens.css")
        └── importado em apps/web/src/app/globals.css
              @import "../../../../packages/ui/src/tokens.css";
              (caminho relativo — @tailwindcss/postcss não resolve symlinks de workspace)
```

### Categorias de tokens

| Categoria   | Exemplos                                                       |
|-------------|----------------------------------------------------------------|
| Tipografia  | `--font-sans`, `--text-h1..h4`, `--text-lead`, `--text-eyebrow` |
| Radii       | `--radius`, `--radius-sm/md/lg/xl/2xl/pill`                    |
| Spacing     | `--space-1/2/3/4/6/8/12/16/24`                                 |
| Elevation   | `--shadow-xs/sm/md/lg`, `--shadow-primary-glow`                |
| Cores       | `--background`, `--foreground`, `--primary`, `--muted`, `--border`, `--destructive`, `--chart-1..5` |

### Paleta semântica (light / dark)

Os tokens de cor são declarados em `:root` (light) e `.dark { }` dentro de `tokens.css`, usando `oklch()` para máxima consistência de contraste. **Nunca use hex diretamente** — adicione um token semântico em `tokens.css`.

> **Tipografia**: `tokens.css` declara `--font-sans: "Inter", system-ui, ...` mas **não carrega** a fonte — quem faz isso é `next/font/google` em `apps/web/src/app/layout.tsx`. A variável do token é suficiente para componentes referenciarem a fonte; o carregamento fica no consumidor.

## Adicionando um componente novo

### Pré-requisito: é um primitivo ou uma composição?

- **Composição** de primitivos existentes → crie em `apps/web/src/components/<feature>/`, não em `packages/ui`.
- **Primitivo novo** que outros apps vão reutilizar → siga os passos abaixo.

### Passo a passo

1. **Instale a dep Radix** (se aplicável):
   ```bash
   npm install @radix-ui/react-<nome> -w packages/ui
   ```

2. **Crie o arquivo** `packages/ui/src/<nome>.tsx`:
   ```tsx
   "use client";
   import * as React from "react";
   import * as Primitive from "@radix-ui/react-<nome>";
   import { cn } from "./utils";

   const MeuComponente = React.forwardRef<...>(({ className, ...props }, ref) => (
     <Primitive.Root ref={ref} className={cn("...", className)} {...props} />
   ));
   MeuComponente.displayName = Primitive.Root.displayName;

   export { MeuComponente };
   ```

3. **Exporte do `index.ts`**:
   ```ts
   export { MeuComponente } from "./meu-componente";
   ```

4. **Adicione ao `package.json`** (`exports` + `dependencies`):
   ```json
   {
     "exports": { "./meu-componente": "./src/meu-componente.tsx" },
     "dependencies": { "@radix-ui/react-<nome>": "^x.y.z" }
   }
   ```

5. **Migre os usos** no app:
   ```bash
   find apps/ -type f -name "*.tsx" | xargs sed -i \
     's|from "@/components/ui/meu-componente"|from "@dosc-syspro/ui"|g'
   ```

6. **Lint e commit**:
   ```bash
   cd apps/web && npx next lint
   git add packages/ui/ && git commit -m "feat(ui): add MeuComponente"
   ```

## Uso correto vs. incorreto

```tsx
// ✅ CORRETO
import { Button, Card, Table } from "@dosc-syspro/ui";

// ❌ ERRADO — nunca importe de @/components/ui/
import { Button } from "@/components/ui/button";

// ❌ ERRADO — nunca redeclare um primitivo localmente se ele já existe
const MyButton = ({ children }) => <button className="bg-black text-white">{children}</button>;
```
