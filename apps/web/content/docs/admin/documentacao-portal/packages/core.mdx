# Package: @dosc-syspro/core

> Configurações globais de negócio e lógica de domínio transversal compartilhada entre API e Web.
> Atualizado em: 2026-05-22

---

## Responsabilidade

`@dosc-syspro/core` contém lógicas transversais de configuração que precisam ser compartilhadas diretamente entre o backend (`apps/api`) e o frontend (`apps/web`):

- **Controle RBAC**: Definição de roles e seus respectivos labels de UI legíveis para o usuário.
- **Bloqueio de Contratos**: Regras e motivos globais para bloqueio preventivo de clientes.
- **Configurações Globais**: Regras organizacionais estáticas.

> 💡 **Nota Importante sobre Arquitetura**: Anteriormente, este pacote gerenciava de forma centralizada entidades legadas como `ticket.entity.ts`. Como parte da modularização e do alinhamento terminológico de **Tickets**, todas as lógicas ativas de tickets, matrizes de transição de estado e cálculos dinâmicos de SLA foram totalmente extraídos e encapsulados nos pacotes modulares de feature **`@dosc-syspro/tickets-domain`** e **`@dosc-syspro/tickets-infra`**, garantindo um core mais limpo e livre de acoplamento técnico.

## Estrutura Física Real

A pasta física do pacote `@dosc-syspro/core` está organizada da seguinte forma:

```
packages/core/src/
├── config/                      ← Configurações transversais de roles e sistemas
│   └── index.ts
│
├── entities/                    ← Entidades remanescentes globais e utilitárias
│   └── index.ts
│
├── services/                    ← Serviços transversais de suporte
│   ├── rbac.ts                        ← Serviços e validadores de escopo de Role
│   └── index.ts
│
└── index.ts                     ← Ponto central de exportação do pacote
```

## Recursos Disponíveis em `@dosc-syspro/core`

### 1. RBAC — Labels de Roles de Usuário

Mapeamento centralizado de roles internas do banco de dados para exibições amigáveis no frontend (utilizado em cabeçalhos, formulários de perfil e listas de usuários):

```typescript
import { Role } from "@prisma/client"

export const roleLabels: Record<Role, string> = {
  ADMIN:         "Administrador",
  SUPORTE:       "Suporte Técnico",
  CLIENTE_ADMIN: "Administrador do Cliente",
  CLIENTE_USER:  "Usuário do Cliente",
  DEVELOPER:     "Desenvolvedor",
}
```

### 2. Bloqueio de Contratos

Estrutura de tipagem que define os motivos técnicos que geram a suspensão ou bloqueio preventivo de contratos ativos de empresas:

```typescript
export type ContractBlockReason =
  | 'PAYMENT_OVERDUE'
  | 'MANUAL_SUSPENSION'
  | 'TRIAL_EXPIRED'
  | 'VIOLATION'
```

## Relação com Pacotes de Features

Ao refatorar regras de negócio específicas para o core, prefira as diretivas da **Arquitetura Hexagonal**:
*   Lógica exclusiva de **Tickets** ➔ deve viver sob `@dosc-syspro/tickets-domain`.
*   Lógica exclusiva de **Leads/Vendas** ➔ deve viver sob `@dosc-syspro/crm-domain`.
*   Lógica exclusiva de **Tarefas de Equipe** ➔ deve viver sob `@dosc-syspro/tarefas-domain`.
