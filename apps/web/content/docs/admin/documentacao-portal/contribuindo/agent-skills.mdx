# Agent Skills

<Callout type="info">
  Estas skills foram adaptadas da matriz usada no projeto Cadens e ajustadas para a arquitetura real do Portal Trilink / Syspro.
</Callout>

## Objetivo

Padronizar como agentes de IA e desenvolvedores atacam problemas de arquitetura, duplicacao, autorizacao, banco, UI e documentacao neste monorepo.

As skills locais vivem em `/.agents/skills`. Elas nao substituem authorities oficiais do ecossistema; servem para registrar as regras especificas deste repositorio.

## Authorities complementares

Quando o ambiente do agente disponibilizar skills oficiais ou curadas, combine com a skill local mais proxima:

| Tema | Authority externa | Combinar com |
| --- | --- | --- |
| React / Next.js | `vercel-react-best-practices` | `syspro-ui-system` |
| Composicao de componentes | `vercel-composition-patterns` | `syspro-ui-system` |
| Acessibilidade e UX | `web-design-guidelines` | `syspro-ui-system` |
| Monorepo e pipelines | `turborepo` | `syspro-monorepo-governance` |
| Prisma ORM | `prisma-cli`, `prisma-client-api` | `syspro-prisma-migrations` |
| Seguranca e threat modeling | `security-best-practices` | `syspro-auth-authorization`, `syspro-worker-reliability` |

## Como usar

1. Comece por `syspro-agent-governance` em tarefas amplas ou ambiguas.
2. Empilhe a skill local da area dominante do problema.
3. Se existir uma authority externa mais forte para a tecnologia envolvida, combine as duas.
4. Verifique sempre se a documentacao do portal ainda representa o codigo real depois da mudanca.

## Manutencao

Sempre que uma skill nova entrar ou uma authority mudar de escopo:

1. atualize o arquivo correspondente em `.agents/skills`
2. atualize esta pagina
3. revise se a navegacao de `Contribuindo` ainda aponta para a referencia certa
