## Responsabilidade

O `apps/web` e o frontend principal do portal Trilink. Ele combina:
- portal administrativo autenticado
- documentacao interna e externa em MDX
- rotas proxy para o backend NestJS
- interfaces de configuracao, tickets, infraestrutura e tarefas

---

## Estrutura principal

```text
apps/web/
|-- src/
|   |-- app/
|   |   |-- (autenticacao)/
|   |   |-- (platform)/portal/
|   |   |-- (site)/
|   |   |-- api/

|-- content/docs/
|   |-- admin/

```

## API Routes de proxy

As rotas em `src/app/api/` fazem proxy para o backend NestJS.

Grupos principais:
- `/api/platform/**`
- `/api/tickets/**`
- `/api/remote/**`
- `/api/remote-admin/**`
- `/api/tax/**`
- `/api/crm/**`
- `/api/search`
- `/api/sefaz/**`

No modulo de tarefas, os pontos importantes no web sao:

```text
/api/platform/settings/tarefas
/portal/tarefas
```

Na interface atual de tarefas, o portal expoe:
- filtro por tipo e status com URL sincronizada
- paginacao da fila operacional
- criacao manual de tarefa avulsa
- abertura opcional de tarefa ao concluir um ticket
- link visual entre tarefa e ticket quando houver origem operacional

## Observacoes de padrao

- URLs de filtro devem refletir estado da tela.
- Listagens grandes devem usar paginacao server-driven.
- Features de dominio devem usar nomenclatura atual do modulo, evitando aliases legados quando a migracao ja foi concluida.
