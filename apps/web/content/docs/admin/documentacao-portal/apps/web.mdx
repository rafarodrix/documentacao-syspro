## Responsabilidade

O `apps/web` e o frontend principal do portal Trilink. Ele combina:
- portal administrativo autenticado
- documentacao interna e externa em MDX
- rotas proxy para o backend NestJS
- interfaces de configuracao, tickets, infraestrutura e tarefas

## API Routes de proxy

As rotas em `src/app/api/` fazem proxy para o backend NestJS.

Grupos principais:
- `/api/platform/**`
- `/api/tickets/**`
- `/api/remote/**`
- `/api/remote-admin/**`
- `/api/tax/**`
- `/api/crm/**`
- `/api/search`
- `/api/sefaz/**`

No modulo de tarefas, os pontos importantes no web sao:

```text
/api/platform/settings/tarefas
/portal/tarefas
```

Na interface atual de tarefas, o portal expoe:
- filtro por tipo e status com URL sincronizada
- paginacao da fila operacional
- criacao manual de tarefa avulsa
- abertura opcional de tarefa ao concluir um ticket
- link visual entre tarefa e ticket quando houver origem operacional

## Deploy proprio

O `apps/web` agora suporta self-hosting com `Next.js standalone`.

Arquivos operacionais:

```text
apps/web/deploy/Dockerfile
apps/web/deploy/docker-compose.yml
```

Fluxo esperado:

1. build da imagem com contexto na raiz do monorepo
2. injeção das variaveis publicas no momento do build/deploy
3. publicacao atras de proxy reverso
4. healthcheck em `/api/health`

### Variaveis criticas

- `NEXT_PUBLIC_APP_URL`
- `NEXT_PUBLIC_WEB_URL`
- `NEXT_PUBLIC_SITE_URL`
- `NEXT_PUBLIC_AUTH_BASE_URL`
- `NEXT_PUBLIC_API_URL`
- `APP_BACKEND_API_URL`
- `BETTER_AUTH_SECRET`
- `INTERNAL_API_KEY`

> As variaveis `NEXT_PUBLIC_*` precisam estar corretas no build, porque o Next.js as incorpora no bundle entregue ao navegador.

### Headers obrigatorios do proxy

O proxy reverso deve repassar:

- `Host`
- `X-Forwarded-Host`
- `X-Forwarded-Proto`

O frontend usa esses headers para resolver origem efetiva, cookies, redirects e chamadas SSR autenticadas.

## Observacoes de padrao

- URLs de filtro devem refletir estado da tela.
- Listagens grandes devem usar paginacao server-driven.
- Features de dominio devem usar nomenclatura atual do modulo, evitando aliases legados quando a migracao ja foi concluida.
