# API — Módulo remote-admin

> `apps/api/src/modules/remote-admin/` · Atualizado em: 2026-05-05

---

## Responsabilidade

O módulo `remote-admin` expõe os endpoints REST que:
1. Recebem chamadas do agente Go (heartbeat, sync, ack, bootstrap, discover)
2. Expõem operações administrativas do portal (iniciar sessão, listar hosts, etc.)
3. Fazem a ponte entre os controllers HTTP e o `@dosc-syspro/remote-domain`

---

## Estrutura de arquivos

```
src/modules/remote-admin/
├── remote-admin.controller.ts     ← endpoints administrativos (portal → hosts)
├── remote-admin.module.ts         ← módulo NestJS
├── remote-admin.service.ts        ← orquestra use cases do remote-domain
├── remote-portal.controller.ts    ← endpoints do portal (sessões, address book)
├── remote-public.controller.ts    ← endpoints públicos do agente (sem auth de usuário)
├── remote-public.service.ts       ← processa heartbeat, sync, ack, bootstrap
├── remote-procedure-runner.ts     ← executa procedimentos remotos privilegiados
└── support/
    ├── operational-status.ts      ← resolve status operacional do host (ONLINE/OFFLINE/etc.)
    ├── remote-admin.types.ts      ← tipos internos do módulo
    ├── remote-host.queries.ts     ← queries reutilizáveis de hosts
    ├── report-queries.ts          ← queries para relatórios de fleet
    ├── scope.ts                   ← resolução de escopo (RemoteTenantScope)
    └── session-queries.ts         ← queries de sessões remotas
```

## Controllers e seus endpoints

### `remote-public.controller.ts` — para o agente Go

Endpoints sem autenticação de usuário, protegidos pelo `agentToken`:

| Método | Path                                     | Ação                                     |
|--------|------------------------------------------|------------------------------------------|
| POST   | `/api/remote/agents/discover`            | Discovery (primeiro contato)             |
| POST   | `/api/remote/rustdesk/bootstrap`         | Registro e obtenção de agentToken        |
| POST   | `/api/remote/rustdesk/heartbeat`         | Heartbeat periódico do agente            |
| POST   | `/api/remote/rustdesk/sync`              | Sync de estado e relatórios do sistema   |
| POST   | `/api/remote/rustdesk/ack`               | Confirmação de comandos executados       |

**Proteção anti-abuso:**
- Rate limit em `/api/remote/agents/discover`: 5 req/min por IP
- Resposta 429 quando excede a janela

### `remote-admin.controller.ts` — operações administrativas

Requer autenticação de usuário e permissão `remote:manage`:

| Método | Path                                               | Ação                                     |
|--------|----------------------------------------------------|------------------------------------------|
| GET    | `/api/remote/hosts`                               | Lista hosts com status operacional       |
| GET    | `/api/remote/hosts/:id`                           | Detalhe de um host                       |
| POST   | `/api/remote/hosts/:id/actions`                   | Enfileira comando no host                |
| GET    | `/api/remote/hosts/:id/events`                    | Eventos recentes do host                 |
| GET    | `/api/remote/hosts/:id/ack-events`                | Histórico de ACKs                        |
| GET    | `/api/remote/hosts/fleet-stats`                   | Estatísticas da frota                    |
| GET    | `/api/remote/discovered-hosts`                    | Hosts descobertos (pendente de link)     |
| POST   | `/api/remote/discovered-hosts/:id/link`           | Vincula host descoberto a empresa/host   |
| POST   | `/api/remote/hosts/:id/agent-token`               | Rotação de agentToken                    |

### `remote-portal.controller.ts` — sessões e address book

| Método | Path                                               | Ação                                     |
|--------|----------------------------------------------------|------------------------------------------|
| GET    | `/api/remote/sessions`                            | Lista sessões ativas                     |
| POST   | `/api/remote/sessions/:id/start`                  | Inicia sessão de acesso remoto           |
| POST   | `/api/remote/sessions/:id/stop`                   | Encerra sessão                           |
| GET    | `/api/remote/rustdesk/address-book`               | Address book para RustDesk client        |
| GET    | `/api/remote/rustdesk/address-book/credentials`   | Credenciais de address book              |
| POST   | `/api/remote/rustdesk/address-book/credentials`   | Cria credencial                          |

## Status operacional dos hosts

Calculado em `support/operational-status.ts`:

| Status           | Critério                                                      |
|------------------|---------------------------------------------------------------|
| `ONLINE`         | Último heartbeat < 2 minutos                                  |
| `RECENT`         | Último heartbeat entre 2 e 15 minutos                         |
| `OFFLINE`        | Último heartbeat > 15 minutos                                 |
| `MISCONFIGURED`  | agentToken ausente, rustdeskId inválido ou config corrompida  |
| `SESSION_BUSY`   | Há sessão ativa em andamento                                  |

## Fila de comandos (RemoteAgentCommand)

O portal pode enfileirar comandos que o agente executa no próximo ciclo:

| Comando                   | Efeito no agente                                      |
|---------------------------|-------------------------------------------------------|
| `REAPPLY_ALIAS`           | Reconfigura o alias/ID do RustDesk                    |
| `REAPPLY_CONFIG`          | Reaplica o arquivo de configuração do RustDesk        |
| `UPGRADE_CLIENT`          | Dispara atualização do cliente RustDesk               |
| `ROTATE_TOKEN_REQUIRED`   | Força o agente a solicitar novo agentToken            |

Comandos são persistidos no modelo `RemoteAgentCommand` com status PENDING → ACKNOWLEDGED.

## remote-procedure-runner.ts

Executa procedimentos privilegiados no contexto do agente (ex: restart de serviços).
Requer permissão especial e escopo correto de empresa.
Conecta ao módulo `remote-admin` via `remote-portal.controller`.
