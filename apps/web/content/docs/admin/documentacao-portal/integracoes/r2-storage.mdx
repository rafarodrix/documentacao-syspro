# Integração: Cloudflare R2 (Object Storage)

> Armazenamento de objetos compatível com S3. Atualizado em: 2026-05-05

---

## O que é

Cloudflare R2 é usado como object storage para arquivos do portal. É compatível com a API S3 da AWS, então o SDK AWS é usado para integração.

---

## Arquivos relevantes

| Arquivo                                              | Responsabilidade            |
|------------------------------------------------------|-----------------------------|
| `apps/api/src/modules/integrations/storage/r2-storage.service.ts` | Upload, download, URLs |
| `packages/config/src/` (r2-related)                 | Leitura de env vars         |

---

## Configuração

```env
R2_BUCKET_NAME=trilink-prod
R2_ENDPOINT=https://<account-id>.r2.cloudflarestorage.com
R2_ACCESS_KEY_ID=<access-key>
R2_SECRET_ACCESS_KEY=<secret-key>
R2_PUBLIC_BASE_URL=https://cdn.trilink.com.br  # opcional: URL pública do bucket
```

Se `R2_PUBLIC_BASE_URL` estiver definido, arquivos públicos usam URL simples.
Caso contrário, URLs assinadas com TTL são geradas para acesso temporário.

---

## Uso no código

```typescript
// Injetado via DI no NestJS
constructor(private readonly r2Storage: R2StorageService) {}

// Upload
const key = await this.r2Storage.upload({
  key: `documentos/empresa-${companyId}/${filename}`,
  buffer: fileBuffer,
  contentType: 'application/pdf',
  metadata: { uploadedBy: userId },
})

// URL de acesso
const url = await this.r2Storage.getUrl(key, { ttl: 3600 }) // 1 hora
```

---

## Onde é usado

| Módulo          | Uso                                              | Visibilidade    |
|-----------------|--------------------------------------------------|-----------------|
| `documentos`    | Documentos de empresa (contratos, certificados) | Signed URL      |
| `tickets`       | Anexos de chamados técnicos                      | Signed URL      |
| `messaging`     | Mídia de mensagens WhatsApp (imagens, docs)      | Signed URL      |
| `backup`        | Backups Firebird do agente Go                    | Signed URL      |

---

## Estrutura de keys no bucket

Convenção de organização no bucket:

```
documentos/<companyId>/<filename>
tickets/<ticketId>/attachments/<filename>
messaging/<conversationId>/<messageId>/<filename>
backups/<hostId>/<date>/<filename>
```
