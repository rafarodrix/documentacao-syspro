# Integração: Evolution API (WhatsApp)

> Mensagens WhatsApp via Evolution API (fork open-source do Baileys).
> Atualizado em: 2026-07-09

---

## O que é

A Evolution API é o adaptador WhatsApp Web usado para enviar e receber mensagens WhatsApp. A integração permite:
- Receber mensagens de clientes via WhatsApp
- Enviar mensagens de resposta pelo portal ou Chatwoot
- Gerenciar conexões (QR code, status)
- Automações de atendimento

## Configuração

```bash
EVOLUTION_API_URL=https://evolution.example.com
EVOLUTION_API_KEY=api_key_aqui
```

Configuracao da instancia e do webhook em `/portal/configuracoes?tab=evolution`.

---

## Webhook: Evolution → Portal

**Endpoint:** `POST /api/webhooks/evolution`

> [!WARNING]
> O webhook da Evolution deve apontar para o backend NestJS em `backend.<dominio>`, nao para `api.<dominio>`.
> Exemplo de producao Trilink: `https://backend.trilinksoftware.com.br/api/webhooks/evolution`.

Eventos processados:

| Evento Evolution         | Ação no portal                                      |
|--------------------------|-----------------------------------------------------|
| `messages.upsert`        | Nova mensagem ou atualização → process-incoming-message.usecase |
| `connection.update`      | Atualiza status da conexão WhatsApp                 |
| `qrcode.updated`         | Novo QR code para conexão da instância              |
| `messages.delete`        | Marca mensagem como deletada                        |

## Fluxo de mensagem recebida

```
Cliente WhatsApp
    │ mensagem
    ▼
Evolution API
    │ webhook POST
    ▼
evolution-webhook.controller
    │
    ▼
integration-webhook-dedup.service
    │ verifica messageId duplicado
    ▼
process-incoming-message.usecase
    ├── resolve empresa pelo número de destino
    ├── busca/cria CompanyContact (source: WHATSAPP)
    ├── busca/cria Conversation (channel: WHATSAPP)
    ├── persiste ConversationMessage
    └── encaminha para Chatwoot (se integração ativa)
```

## Envio de mensagens

**Endpoint:** `POST /api/platform/integrations/evolution/messages/send`

```typescript
// Payload
{
  instanceName: 'minha-instancia',
  number: '5511987654321',
  text: 'Olá, como posso ajudar?',
}
```

O `EvolutionClient` define o port consumido pelo app. Hoje a chamada HTTP fica em `evolution-go.client.ts`, o que permite trocar a implementacao para uma API oficial depois sem reespalhar rotas ou bindings pelos modulos consumidores.

## Gerenciamento de conexão

Disponível em `/portal/configuracoes?tab=evolution`:
- Exibir QR code para parear WhatsApp
- Status da conexão (conectado/desconectado)
- Reconectar instância
- Persistir o payload efetivo de `POST /instance/connect`

**Endpoints:**
- `GET /api/platform/settings/evolution/status` — status da conexão
- `POST /api/platform/settings/evolution/qrcode` — solicita conexão e aguarda QR code

Campos operacionais persistidos hoje:
- `webhookUrl`
- `subscribe`
- `immediate`
- `phone`
- `rabbitmqEnable`
- `websocketEnable`
- `natsEnable`
- `instance`
- `instanceId`
- `instanceToken`

## Normalização de números

Antes de processar mensagens, o número WhatsApp passa por normalização de domínio:

- Remove caracteres não numéricos
- Normaliza DDI Brasil (55)
- `newsletter` é tratado como domínio válido (grupos de transmissão)

## Automações

O módulo `automation` permite configurar respostas automáticas baseadas em triggers:

```typescript
// Exemplo de automação
{
  trigger: 'FIRST_MESSAGE',
  action: 'SEND_MESSAGE',
  message: 'Olá! Recebi sua mensagem e responderemos em breve.',
  instanceName: 'minha-instancia',
}
```

Configurado em `/portal/configuracoes?tab=automacoes`.
