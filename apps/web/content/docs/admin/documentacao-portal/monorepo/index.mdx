# Visão Geral do Monorepo

> Stack: **npm workspaces** · Atualizado em: 2026-05-05

---

## O que é este projeto

O **Trilink Portal** é uma plataforma de gestão integrada para empresas que utilizam o ERP Syspro. Combina:

- **Portal de negócio**: gestão de empresas, contratos, contatos, documentos, SEFAZ/NFe
- **Suporte técnico**: ticketing, CRM, automação com WhatsApp (Evolution) e Chatwoot
- **Infraestrutura remota**: gerenciamento de máquinas Windows via RustDesk, com agentes Go
- **Documentação interna**: base de conhecimento técnico e manuais para clientes

---

## Estrutura do monorepo

```
/
├── apps/
│   ├── api/        NestJS — backend principal (REST + tRPC)
│   ├── web/        Next.js 15 — frontend do portal
│   ├── agent/      Go + Wails — agente desktop Windows
│   └── mobile/     TypeScript — placeholder (não implementado)
│
├── packages/
│   ├── contracts/      Zod schemas e tipos TypeScript compartilhados
│   ├── remote-domain/  Lógica de domínio do módulo remoto (ports & adapters)
│   ├── remote-infra/   Implementações das ports do remote-domain
│   ├── database/       Prisma ORM (schema PostgreSQL + client)
│   ├── shared/         Utilitários: formatadores, logger, auth helpers
│   ├── core/           Entidades de negócio: tickets, releases, RBAC config
│   ├── ui/             Componentes React (Radix UI)
│   └── config/         Leitura de variáveis de ambiente com validação Zod
│
├── scripts/            Scripts de manutenção e verificação
└── package.json        Workspace root
```

## Stack tecnológica por app

### `apps/api` — Backend NestJS

| Camada         | Tecnologia                    |
|----------------|-------------------------------|
| Framework      | NestJS 11                     |
| Linguagem      | TypeScript 5.8                |
| ORM            | Prisma 5.22 + PostgreSQL      |
| Autenticação   | Better Auth 1.3               |
| API type-safe  | tRPC 11 (servidor)            |
| Object Storage | AWS SDK S3 → Cloudflare R2    |
| Email          | Nodemailer 8                  |
| Testes         | Vitest                        |
| Build          | Webpack (custom config)       |

### `apps/web` — Frontend Next.js

| Camada         | Tecnologia                    |
|----------------|-------------------------------|
| Framework      | Next.js 15 (App Router)       |
| Linguagem      | TypeScript 5.8 / React 19     |
| Estilo         | Tailwind CSS + Radix UI        |
| API type-safe  | tRPC Client 11                |
| Documentação   | Fumadocs (MDX)                |
| Gráficos       | ApexCharts                    |
| Testes unit.   | Vitest                        |
| Testes E2E     | Playwright                    |

### `apps/agent` — Agente Windows (Go)

| Camada         | Tecnologia                    |
|----------------|-------------------------------|
| Linguagem      | Go 1.26                       |
| UI desktop     | Wails v2 (WebView2 + React)   |
| Acesso remoto  | RustDesk (integração interna) |
| Build          | wails build (Windows)         |
| IPC local      | Named pipe (Windows) / Unix socket |

### `apps/mobile` — Placeholder

Arquivo TypeScript estático. Estrutura preparada para Expo/React Native no futuro. Não implementado.

## Scripts npm workspaces

O projeto usa **npm workspaces** diretamente — sem Turborepo. Os comandos seguem o padrão:

```bash
# Rodar um script em um workspace específico
npm run dev -w web
npm run dev -w api
npm run build -w web
npm run build -w api

# Rodar testes
npm run test -w web
npm run test:e2e -w web   # Playwright E2E
```

## Variáveis de ambiente

Cada app tem seu próprio `.env.local`. As principais:

| Variável              | Usada por       | Descrição                            |
|-----------------------|-----------------|--------------------------------------|
| `DATABASE_URL`        | api, database   | PostgreSQL connection string          |
| `DIRECT_URL`          | database        | Direct URL sem PgBouncer (migrations) |
| `BETTER_AUTH_SECRET`  | api             | Secret do Better Auth                 |
| `R2_BUCKET_NAME`      | api             | Nome do bucket Cloudflare R2          |
| `R2_ENDPOINT`         | api             | Endpoint S3-compat do R2              |
| `CHATWOOT_API_KEY`    | api             | Token da API Chatwoot                 |
| `EVOLUTION_API_URL`   | api             | URL da Evolution API (WhatsApp)       |
| `NEXT_PUBLIC_API_URL` | web             | URL base da API para o frontend       |

## Convenção de nomes de packages

Todos os packages internos usam o escopo `@dosc-syspro/`:

```
@dosc-syspro/contracts
@dosc-syspro/database
@dosc-syspro/shared
...
```

Os apps **não** usam escopo — são referenciados pelo workspace name simples (`api`, `web`, `agent`).
