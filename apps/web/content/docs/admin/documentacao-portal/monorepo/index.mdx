# Visão Geral do Monorepo

> Stack: **npm workspaces + Turborepo** · Atualizado em: 2026-05-22

---

## O que é este projeto

O **Trilink Portal** é uma plataforma de gestão integrada para empresas que utilizam o ERP Syspro. Combina:

- **Portal de negócio**: gestão de empresas, contratos, contatos, documentos, SEFAZ/NFe
- **Suporte técnico**: ticketing, CRM, automação com WhatsApp (Evolution) e Chatwoot
- **Infraestrutura remota**: gerenciamento de máquinas Windows via RustDesk, com agentes Go
- **Documentação interna**: base de conhecimento técnico e manuais para clientes

## Estrutura do monorepo

```
/
├── apps/
│   ├── api/        NestJS — backend principal (@dosc-syspro/app-api)
│   ├── web/        Next.js 15 — frontend do portal (@dosc-syspro/web)
│   ├── agent/      Go + Wails — agente desktop Windows
│   └── mobile/     TypeScript/Expo — aplicativo mobile (@dosc-syspro/app-mobile)
│
├── packages/
│   ├── contracts/  Zod schemas e tipos TypeScript compartilhados (@dosc-syspro/contracts)
│   ├── database/   Prisma ORM (schema PostgreSQL + client) (@dosc-syspro/database)
│   ├── shared/     Utilitários: formatadores, logger, auth helpers (@dosc-syspro/shared)
│   ├── core/       Entidades de negócio e helpers transversais (@dosc-syspro/core)
│   ├── ui/         Design System e componentes React compartilhados (@dosc-syspro/ui)
│   ├── config/     Leitura de variáveis de ambiente com validação Zod (@dosc-syspro/config)
│   │
│   └── features/   Módulos de negócio com arquitetura hexagonal desacoplada
│       ├── crm/          Domínio e Infraestrutura de CRM (leads/oportunidades)
│       ├── remote/       Domínio e Infraestrutura de infra remota (RustDesk/Hosts)
│       ├── tarefas/      Domínio e Infraestrutura de gerenciamento de tarefas/SLA
│       └── tickets/      Domínio e Infraestrutura de suporte e tickets administrativos
│
├── scripts/        Scripts de manutenção, auditoria e verificação
├── turbo.json      Configurações do Turborepo (pipelines e cache)
└── package.json    Workspace root
```

## Packages — mapa de responsabilidades

| Package             | Escopo                                                        | Consumido por                |
|---------------------|---------------------------------------------------------------|------------------------------|
| `@dosc-syspro/contracts`     | Schemas Zod + tipos para todos os domínios             | api, web, pacotes locais     |
| `@dosc-syspro/database`      | PrismaClient gerado + schema                           | api, pacotes de infra        |
| `@dosc-syspro/shared`        | Formatadores, logger, HMAC, rate limit                 | api, web                     |
| `@dosc-syspro/core`          | RBAC, labels de UI, regras globais de negócios         | api, web                     |
| `@dosc-syspro/ui`            | Componentes React base (Design System e shadcn)        | web                          |
| `@dosc-syspro/config`        | Leitura de env vars com Zod                            | api, pacotes de infra        |

> **Nota sobre Modularização Hexagonal**: Toda a lógica de casos de uso, interfaces e infraestrutura específica dos módulos administrativos principais (CRM, Remote Admin, Tarefas, Tickets) vive encapsulada em seus respectivos subpacotes em `packages/features/` (ex: `@dosc-syspro/tickets-domain` e `@dosc-syspro/tickets-infra`).

## Scripts do Turborepo

O projeto utiliza o **Turborepo** para orquestrar tarefas de build, typecheck, lint e testes de maneira paralela e eficiente, aproveitando cache local e remoto.

```bash
# Executar typecheck em todo o monorepo de forma otimizada
npm run typecheck       # Roda "turbo run typecheck"

# Compilar todos os pacotes e aplicações locais
npm run build           # Roda "turbo run build"

# Rodar o servidor de desenvolvimento do frontend
npm run dev             # Filtra e executa o frontend web no Turbo

# Executar testes da API backend
npm run test:api        # Roda a suíte Vitest em @dosc-syspro/app-api

# Executar testes do frontend web
npm run test:web        # Roda a suíte Vitest em @dosc-syspro/web
```

## Convenção de nomes de packages

Todos os packages e aplicações no monorepo usam a nomenclatura oficial corporativa com escopo `@dosc-syspro/`:

```json
// Exemplos de package.json
{ "name": "@dosc-syspro/web" }
{ "name": "@dosc-syspro/app-api" }
{ "name": "@dosc-syspro/app-mobile" }
{ "name": "@dosc-syspro/tickets-domain" }
```
