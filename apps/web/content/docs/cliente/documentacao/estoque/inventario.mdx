# Cadastro de Inventário

## Introdução

A tela de **Cadastro de Inventário** é utilizada para registrar e controlar a contagem física dos produtos em estoque no Syspro ERP.

Neste cadastro, a empresa define uma descrição para o inventário, informa a **data base** da contagem, seleciona o **grupo de estoque** quando aplicável e define o **tipo de produto** que será considerado na geração do inventário.

Após preencher os dados principais, o botão **Importar** carrega os produtos e quantidades do estoque conforme a **data base informada**.

<Callout type="info" title="Objetivo do módulo">
  Permitir a criação de inventários de estoque para conferência física, ajuste operacional e validação das quantidades registradas no sistema.
</Callout>

| Situação | Como o módulo ajuda |
|---|---|
| Contagem física periódica | Permite registrar a posição do estoque em uma data base específica. |
| Fechamento de estoque | Ajuda a comparar o estoque físico com o saldo registrado no sistema. |
| Conferência por grupo de estoque | Permite gerar inventário filtrando produtos por grupo, quando aplicável. |
| Conferência por tipo de produto | Permite selecionar quais tipos de produtos farão parte do inventário. |
| Ajuste operacional | Pode apoiar a identificação de divergências entre estoque físico e sistema. |

## Estrutura da tela

<Tabs items={['Consulta', 'Inclusão/Edição', 'Importação']}>
  <Tab value="Consulta">
    A tela de consulta apresenta os inventários já cadastrados, permitindo localizar registros pela **descrição** e pela **empresa**.

    As principais ações disponíveis são:

    - **Incluir**: cria um novo inventário;
    - **Editar**: altera um inventário existente;
    - **Excluir**: remove um inventário, conforme permissões e regras internas;
    - **Clonar**: copia informações de um inventário existente;
    - **Filtrar**: aplica filtros na listagem;
    - **Consultar**: atualiza ou busca os registros cadastrados.
  </Tab>

  <Tab value="Inclusão/Edição">
    Na inclusão ou edição do inventário, o usuário deve preencher os campos principais, como **Descrição**, **Tipo Custo**, **Tipo Produto** e **Data Base**.

    Também é possível informar **Grupo de Estoque** para limitar o inventário a um grupo específico de produtos.
  </Tab>

  <Tab value="Importação">
    Após preencher os dados principais, clique em **Importar**.

    O sistema carrega os produtos e suas quantidades de estoque conforme a **data base informada**.

    <Callout type="info" title="Importação pela data base">
      Ao clicar em **Importar**, o sistema utiliza a **Data Base** para buscar a posição do estoque naquela data. Essa informação é essencial para que o inventário represente corretamente o período desejado.
    </Callout>
  </Tab>
</Tabs>

# Campos principais

## Descrição

A **Descrição** identifica o inventário dentro do sistema.

Use nomes claros, que facilitem a localização futura.

Exemplos:

- INVENTÁRIO 2025;
- INVENTÁRIO LOJA 01;
- INVENTÁRIO DEPÓSITO CENTRAL;
- INVENTÁRIO FINAL DE ANO.

<Callout type="success" title="Boa prática">
  Utilize uma descrição que indique o período, filial, depósito ou objetivo do inventário.
</Callout>

---

## Tipo Custo

O campo **Tipo Custo** define qual custo será utilizado como referência no inventário, conforme configuração da empresa.

Esse campo pode influenciar relatórios, análises de valorização de estoque e conferências gerenciais.

<Callout type="warning" title="Atenção">
  Antes de salvar ou importar o inventário, confirme com o responsável administrativo qual tipo de custo deve ser utilizado.
</Callout>

---

## Tipo Produto

O campo **Tipo Produto** define quais tipos de produtos serão considerados na geração do inventário.

Use este campo para limitar a importação conforme a necessidade da contagem.

Exemplos de uso:

- inventariar apenas produtos de venda;
- inventariar apenas matéria-prima;
- inventariar produtos de determinado tipo operacional;
- separar inventários por classificação interna.

<Callout type="info" title="Filtro por tipo de produto">
  O **Tipo Produto** deve ser selecionado conforme os tipos que a empresa deseja gerar no inventário.
</Callout>

---

## Grupo de Estoque

O campo **Grupo de Estoque** define qual grupo será considerado no inventário.

Quando preenchido, o sistema tende a gerar o inventário apenas para produtos vinculados ao grupo selecionado, conforme configuração da empresa.

<Callout type="info" title="Uso do grupo de estoque">
  O **Grupo de Estoque** é utilizado para direcionar o inventário para um conjunto específico de produtos.
</Callout>

---

## Data Base

A **Data Base** é uma das informações mais importantes do inventário.

Ela define a data em que o sistema deve considerar a posição do estoque para importar os produtos e quantidades.

Exemplo:

Se a data base for **31/12/2025**, o sistema buscará as quantidades de estoque conforme essa data.

<Callout type="warning" title="Atenção à data base">
  Revise a **Data Base** antes de clicar em **Importar**. Uma data incorreta pode gerar uma posição de estoque diferente da esperada.
</Callout>

---

## Importar Estoque Zero

O campo **Importar Estoque Zero** define se produtos com saldo zerado também serão incluídos no inventário.

| Opção | Comportamento esperado |
|---|---|
| Sim | Inclui produtos mesmo sem saldo em estoque, quando aplicável. |
| Não | Importa apenas produtos com saldo ou movimentação relevante, conforme regra do sistema. |

<Callout type="info" title="Quando usar estoque zero">
  Marque essa opção quando a empresa deseja conferir também produtos sem saldo, por exemplo, para validar ausência física no estoque.
</Callout>

# Relações com outros cadastros

O inventário possui relação direta com os cadastros e movimentações de estoque.

| Cadastro relacionado | Impacto |
|---|---|
| Produto | Define quais itens podem ser importados para o inventário. |
| Grupo de Estoque | Permite filtrar o inventário por agrupamento de produtos. |
| Tipo de Produto | Permite selecionar quais classificações de produto serão consideradas. |
| Estoque | Fornece a quantidade base para conferência. |
| Empresa | Define em qual empresa o inventário será realizado. |
| Relatórios de Estoque | Podem utilizar os dados do inventário para análise e conferência. |

<Callout type="warning" title="Impacto operacional">
  Alterações ou importações incorretas no inventário podem afetar a conferência de estoque e gerar divergências nos relatórios.
</Callout>

# Como cadastrar

<Steps>
  <Step>Acesse **Estoque > Inventário**.</Step>
  <Step>Clique em **Incluir**.</Step>
  <Step>Informe a **Descrição** do inventário.</Step>
  <Step>Selecione o **Tipo Custo**.</Step>
  <Step>Informe a **Data Base** que será usada para buscar a posição do estoque.</Step>
  <Step>Selecione o **Tipo Produto** conforme os produtos que deseja gerar no inventário.</Step>
  <Step>Informe o **Grupo de Estoque**, quando o inventário for limitado a um grupo específico.</Step>
  <Step>Defina se deseja **Importar Estoque Zero**.</Step>
  <Step>Clique em **Salvar** para gravar o cadastro inicial.</Step>
  <Step>Clique em **Importar** para carregar os produtos e quantidades conforme a data base.</Step>
  <Step>Revise os produtos importados na grade.</Step>
</Steps>

<Callout type="success" title="Resultado esperado">
  Após a importação, a grade do inventário deve apresentar os produtos encontrados conforme os filtros selecionados e a posição de estoque da data base.
</Callout>

# Boas práticas

- Crie inventários com descrições padronizadas.
- Revise a data base antes de importar.
- Utilize grupo de estoque quando a contagem for segmentada.
- Utilize tipo de produto para evitar importação de itens desnecessários.
- Evite alterar o inventário sem orientação do responsável pelo estoque.
- Faça a conferência física antes de qualquer ajuste definitivo.
- Mantenha apenas usuários autorizados com permissão de edição.
- Documente divergências encontradas durante a contagem.

<Callout type="success" title="Boa prática">
  Sempre valide a importação em uma pequena amostra de produtos antes de liberar o inventário para conferência completa.
</Callout>

# Problemas comuns

<Accordions type="single">
  <Accordion title="Os produtos não aparecem após importar" id="produtos-nao-aparecem">
    Verifique se a **Data Base** foi preenchida corretamente, se o **Tipo Produto** foi selecionado e se o **Grupo de Estoque** informado possui produtos vinculados.

    Também confira se os produtos possuem saldo na data informada ou se a opção **Importar Estoque Zero** precisa ser marcada.
  </Accordion>

  <Accordion title="O inventário importou produtos errados" id="inventario-importou-produtos-errados">
    Revise o **Tipo Produto** e o **Grupo de Estoque** selecionados.

    Se o grupo de estoque estiver em branco, o sistema pode considerar uma abrangência maior de produtos, conforme configuração da empresa.
  </Accordion>

  <Accordion title="A quantidade importada não confere com o físico" id="quantidade-nao-confere">
    Confira se a **Data Base** corresponde à data real da contagem.

    Movimentações feitas antes ou depois da data base podem alterar a posição esperada do estoque.
  </Accordion>

  <Accordion title="O botão Importar não gera registros" id="botao-importar-nao-gera-registros">
    Verifique permissões do usuário, filtros preenchidos, vínculo dos produtos com grupo/tipo e se existem saldos ou movimentações na data base informada.
  </Accordion>

  <Accordion title="O usuário não consegue editar o inventário" id="usuario-nao-consegue-editar">
    Confirme se o usuário possui permissão para editar inventários.

    Também valide se o inventário já foi utilizado em algum processo que bloqueia alterações, conforme configuração da empresa.
  </Accordion>
</Accordions>

# Resumo

O **Cadastro de Inventário** permite controlar a contagem física dos produtos no Syspro ERP, utilizando filtros como **Grupo de Estoque**, **Tipo Produto** e **Data Base**.

A principal ação da tela é a **Importação**, que carrega os produtos e quantidades conforme a posição de estoque da data base informada.

<Callout type="success" title="Resumo">
  Para um inventário confiável, revise a data base, selecione corretamente o tipo de produto, utilize grupo de estoque quando necessário e valide os produtos importados antes de iniciar a conferência física.
</Callout>
