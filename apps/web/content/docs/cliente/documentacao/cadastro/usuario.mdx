# Cadastro de Usuário

## Introdução

O cadastro de **Usuário** é utilizado para controlar quem pode acessar o **Syspro ERP**, quais empresas pode operar, quais contas pode movimentar e quais permissões estarão disponíveis no sistema.

Essa tela é uma das principais configurações de segurança do ERP. Por meio dela, a empresa vincula o login de acesso a um participante, define senha, perfil de acesso, empresas permitidas, contas bancárias e grupos de conta caixa.

Uma configuração correta evita acessos indevidos, melhora a rastreabilidade das ações e garante que cada colaborador utilize apenas os recursos necessários para sua função.

<Callout type="info" title="Objetivo do módulo">
  Configurar os usuários autorizados a acessar o Syspro ERP, controlando credenciais, permissões, empresas, contas e restrições operacionais.
</Callout>

## Quando utilizar

Utilize o cadastro de **Usuário** sempre que for necessário conceder, alterar, revisar ou bloquear o acesso de uma pessoa ao Syspro ERP.

| Situação | Como o módulo ajuda |
|---|---|
| Criar novo acesso | Permite cadastrar login, senha e vínculo com participante. |
| Definir permissões | Permite vincular o usuário a um perfil de acesso. |
| Liberar empresas ou filiais | Define quais empresas o usuário poderá acessar. |
| Restringir contas bancárias | Controla quais contas o usuário poderá movimentar. |
| Restringir contas caixa | Permite limitar o usuário a grupos específicos de caixa. |
| Alterar senha | Permite ao administrador redefinir a senha do usuário. |
| Bloquear usuário desligado | Permite inativar o usuário e preservar histórico. |
| Revisar segurança | Facilita auditoria de acessos e permissões. |

<Callout type="success" title="Exemplo prático">
  Ao contratar um novo vendedor, cadastre o participante colaborador, crie o usuário, vincule o perfil **Vendedores**, selecione as empresas permitidas e teste o acesso antes de liberar para uso.
</Callout>

# Campos principais

## Participante

O campo **Participante** vincula o usuário a uma pessoa cadastrada no sistema.

Normalmente, esse participante representa o colaborador que utilizará o login.

<Callout type="info" title="Rastreabilidade">
  Vincular o usuário ao participante correto melhora a identificação de ações realizadas no sistema e facilita auditorias internas.
</Callout>

---

## Nome

O campo **Nome** define o login de acesso do usuário.

Recomendações de padrão:

```txt
nome.sobrenome
primeironome.sobrenome
setor.nome
````

Exemplos:

```txt
joao.silva
maria.financeiro
paulo.vendas
```

<Callout type="success" title="Boa prática">
  Use um padrão de login consistente para facilitar suporte, manutenção e identificação dos usuários.
</Callout>

---

## Senha

O campo **Senha** define a credencial de acesso do usuário.

A senha deve ser informada com cuidado e, quando possível, seguir critérios mínimos de segurança definidos pela empresa.

<Callout type="warning" title="Segurança de senha">
  Evite senhas simples, compartilhadas ou reutilizadas. Cada usuário deve possuir uma senha individual.
</Callout>

---

## Perfil de Acesso

O campo **Perfil de Acesso** define as permissões do usuário.

Ele controla quais módulos, telas e ações estarão disponíveis.

Exemplos de perfis:

* Vendedores;
* Financeiro;
* Compras;
* Estoque;
* Gerência;
* Administrador;
* Suporte.

<Callout type="warning" title="Permissão crítica">
  O perfil de acesso deve ser escolhido conforme a função do colaborador. Um perfil incorreto pode liberar acessos indevidos ou impedir atividades necessárias.
</Callout>

---

## Empresa

A aba **Empresa** define quais empresas ou filiais o usuário poderá acessar.

Use essa configuração quando o ambiente possuir mais de uma empresa cadastrada.

<Callout type="info" title="Ambiente multiempresa">
  Em empresas com matriz e filiais, revise cuidadosamente quais unidades o usuário poderá operar.
</Callout>

---

## Contas Bancárias

A aba **Contas Bancárias** define quais contas o usuário poderá movimentar.

Conforme o comportamento informado, se nenhuma conta for selecionada, o usuário poderá ter acesso a **todas** as contas.

<Callout type="warning" title="Atenção às contas bancárias">
  Se a empresa deseja restringir movimentações financeiras, selecione explicitamente as contas permitidas para o usuário.
</Callout>

---

## Grupo Conta Caixa

A aba **Grupo Conta Caixa** restringe o usuário a um grupo específico de contas caixa.

Esse controle é útil para:

* operadores de caixa;
* setores diferentes;
* filiais;
* pontos de venda;
* separação de responsabilidades financeiras.

# Relações com outros cadastros

O cadastro de **Usuário** se relaciona diretamente com outros cadastros de segurança e operação.

| Cadastro ou módulo relacionado | Impacto                                                                                 |
| ------------------------------ | --------------------------------------------------------------------------------------- |
| Participante                   | O usuário deve estar vinculado a uma pessoa cadastrada no sistema.                      |
| Perfil de Acesso               | Define módulos, telas e ações disponíveis ao usuário.                                   |
| Empresa                        | Define empresas ou filiais que o usuário poderá acessar.                                |
| Contas Bancárias               | Controla quais contas o usuário poderá movimentar.                                      |
| Grupo Conta Caixa              | Restringe operações de caixa por grupo.                                                 |

<Callout type="info" title="Cadastro de segurança">

# Como cadastrar

<Steps>
  <Step>
    Acesse **Cadastro > Usuário > Usuário**.
  </Step>

  <Step>
    Clique em **Incluir** para criar um novo usuário.
  </Step>

  <Step>
    No campo **Participante**, selecione o colaborador ou pessoa vinculada ao login.

```
Caso o participante ainda não exista, cadastre-o antes no cadastro de Participantes.
```

  </Step>

  <Step>
    Preencha o campo **Nome** com o login do usuário.

```
Exemplo: **joao.silva**.
```

  </Step>

  <Step>
    Informe a **Senha** inicial do usuário.
  </Step>

  <Step>
    Acesse a aba **Perfil** e selecione o **Perfil de Acesso** adequado para a função do usuário.
  </Step>

  <Step>
    Acesse a aba **Empresa** e selecione as empresas ou filiais que o usuário poderá acessar.

```
Se a empresa usa múltiplas filiais, revise este vínculo com atenção.
```

  </Step>

  <Step>
    Acesse a aba **Contas Bancárias** e selecione as contas que o usuário poderá movimentar, quando aplicável.
  </Step>

  <Step>
    Acesse a aba **Grupo Conta Caixa** e defina o grupo permitido, quando o usuário operar caixa.
  </Step>

  <Step>
    Revise todas as permissões, empresas e restrições financeiras.
  </Step>

  <Step>
    Clique em **Salvar**.
  </Step>

  <Step>
    Realize um teste de login com o usuário ou valide com o colaborador se os menus e rotinas necessários estão disponíveis.
  </Step>
</Steps>

# Validação por perfil

<Tabs items={['Usuário', 'Administrador', 'Suporte']}> <Tab value="Usuário">
O usuário deve conferir:

```
- se consegue acessar o sistema;
- se a senha funciona;
- se os menus necessários aparecem;
- se consegue acessar a empresa correta;
- se consegue executar as rotinas da sua função;
- se alguma mensagem de acesso negado aparece.

Caso falte acesso, deve solicitar validação ao gestor ou administrador.
```

  </Tab>

  <Tab value="Administrador">
    O administrador deve validar:

```
- participante correto;
- login padronizado;
- perfil de acesso adequado;
- empresas corretas;
- contas bancárias permitidas;
- grupo conta caixa correto;
- usuário ativo ou inativo;
- acessos compatíveis com a função;
- ausência de permissões excessivas.

Recomenda-se revisar usuários periodicamente, principalmente após desligamentos ou mudança de função.
```

  </Tab>

  <Tab value="Suporte">
    O suporte deve verificar:

```
- se o usuário está ativo;
- se a senha foi informada corretamente;
- se o participante está vinculado;
- qual perfil de acesso está selecionado;
- quais empresas estão liberadas;
- se há restrição de contas bancárias;
- se há grupo conta caixa configurado;
- se o problema ocorre apenas com um usuário ou com todos do mesmo perfil.

Para análise, solicite nome do usuário, perfil vinculado, empresa acessada e print da mensagem apresentada.
```

  </Tab>
</Tabs>

# Boas práticas

* Cadastre o participante antes de criar o usuário.
* Use login individual para cada colaborador.
* Evite usuários genéricos compartilhados.
* Padronize o campo nome/login.
* Vincule sempre um perfil de acesso coerente com a função.
* Revise empresas liberadas para o usuário.
* Restrinja contas bancárias quando o usuário não deve acessar todas.
* Configure grupo conta caixa para operadores de caixa.
* Inative usuários desligados.
* Evite excluir usuários com histórico.
* Revise acessos periodicamente.
* Não compartilhe senhas entre colaboradores.

<Callout type="success" title="Boa prática de segurança">
  Cada colaborador deve possuir seu próprio usuário. Isso melhora a rastreabilidade e evita dúvidas sobre quem executou determinada ação.
</Callout>

# Checklist antes de salvar

Antes de salvar o cadastro de Usuário, confira:

* participante correto selecionado;
* login preenchido e padronizado;
* senha informada;
* perfil de acesso correto;
* empresas liberadas revisadas;
* contas bancárias selecionadas, quando houver restrição;
* regra de contas bancárias compreendida;
* grupo conta caixa configurado, quando aplicável;
* usuário ativo apenas quando deve acessar o sistema;
* permissões compatíveis com a função;
* teste de acesso realizado;
* usuário não está duplicado;
* usuário não é compartilhado entre colaboradores.

# Problemas comuns

<Accordions type="single">
  <Accordion title="O usuário não consegue entrar no sistema" id="usuario-nao-entra">
    Verifique se o nome/login e a senha foram digitados corretamente.

```
Também confirme se o usuário está ativo e se o cadastro foi salvo.
```

  </Accordion>

  <Accordion title="O usuário esqueceu a senha" id="usuario-esqueceu-senha">
    O administrador pode acessar o cadastro do usuário e definir uma nova senha no campo **Senha**.

```
Após a alteração, oriente o usuário a acessar novamente o sistema.
```

  </Accordion>

  <Accordion title="O usuário não vê um menu ou relatório" id="usuario-nao-ve-menu">
    Verifique o **Perfil de Acesso** vinculado ao usuário.

```
O perfil precisa conter o módulo e a ação necessária para acessar a funcionalidade.
```

  </Accordion>

  <Accordion title="O usuário não consegue acessar uma empresa" id="usuario-nao-acessa-empresa">
    Confira a aba **Empresa** no cadastro do usuário.

```
Se a empresa ou filial não estiver liberada, o usuário pode não conseguir acessá-la.
```

  </Accordion>

  <Accordion title="O usuário não consegue movimentar uma conta bancária" id="usuario-nao-movimenta-conta">
    Verifique a aba **Contas Bancárias**.

```
Confirme se a conta desejada está liberada para o usuário ou se a política da empresa permite acesso geral quando nenhuma conta está selecionada.
```

  </Accordion>

  <Accordion title="O usuário está acessando contas que não deveria" id="usuario-acessa-contas-indevidas">
    Revise a aba **Contas Bancárias**.

```
Se nenhuma conta for selecionada e o sistema considerar isso como acesso a todas, selecione explicitamente apenas as contas permitidas.
```

  </Accordion>

  <Accordion title="O usuário não consegue operar o caixa correto" id="usuario-nao-opera-caixa">
    Verifique a aba **Grupo Conta Caixa**.

```
O usuário deve estar vinculado ao grupo correspondente à sua operação.
```

  </Accordion>

  <Accordion title="O colaborador foi desligado" id="colaborador-desligado">
    Acesse o cadastro do usuário e realize a **inativação**.

```
Evite excluir o usuário para preservar histórico e rastreabilidade.
```

  </Accordion>
</Accordions>

    Sim, é recomendado vincular o usuário ao participante correspondente.

```
Esse vínculo ajuda na identificação, organização e rastreabilidade das ações.
```

  </Accordion>

  <Accordion title="Qual é a diferença entre usuário e perfil de acesso?" id="usuario-vs-perfil">
    **Usuário** é o login individual de acesso ao sistema.

```
**Perfil de Acesso** é o conjunto de permissões aplicado ao usuário.
```

  </Accordion>

  <Accordion title="Um usuário pode ter acesso a mais de uma empresa?" id="usuario-varias-empresas">
    Sim. Na aba **Empresa**, selecione todas as empresas ou filiais que o usuário poderá acessar.
  </Accordion>

  <Accordion title="O que acontece se nenhuma empresa for selecionada?" id="nenhuma-empresa-selecionada">
    Conforme a regra informada, se nenhuma empresa for selecionada, o usuário pode não ter acesso a nenhuma empresa.

```
Por isso, revise essa aba antes de liberar o usuário.
```

  </Accordion>

  <Accordion title="O que acontece se nenhuma conta bancária for selecionada?" id="nenhuma-conta-bancaria">
    Conforme a regra informada, se nenhuma conta bancária for selecionada, o usuário poderá ter acesso a todas as contas.

```
Caso a empresa deseje restringir acesso, selecione explicitamente as contas permitidas.
```

  </Accordion>

  <Accordion title="O que fazer se o usuário esquecer a senha?" id="esqueceu-senha-faq">
    O administrador pode acessar o cadastro do usuário, alterar o campo **Senha** e salvar uma nova senha.
  </Accordion>

  <Accordion title="Como bloquear o acesso de um usuário desligado?" id="bloquear-usuario-desligado">
    A melhor prática é inativar o usuário.

```
Assim, o acesso é bloqueado e o histórico permanece preservado.
```

  </Accordion>

  <Accordion title="O que devo conferir antes de chamar o suporte?" id="antes-de-chamar-suporte-usuario">
    Confira se o usuário está ativo, se a senha foi redefinida corretamente, se o perfil está vinculado, se a empresa foi liberada e se as contas bancárias ou grupo conta caixa estão configurados.

```
Se possível, envie print da mensagem de erro e informe o login do usuário.
```

  </Accordion>
</Accordions>

# Resumo

O cadastro de **Usuário** é responsável por controlar o acesso ao Syspro ERP.

Ele define credenciais, vínculo com participante, perfil de acesso, empresas permitidas, contas bancárias e grupo conta caixa. Por isso, deve ser configurado com atenção e revisado sempre que houver contratação, mudança de função ou desligamento de colaboradores.

<Callout type="success" title="Resumo">
  Crie usuários individuais, vincule o participante correto, selecione o perfil adequado, revise empresas e contas permitidas e inative usuários que não devem mais acessar o sistema.
</Callout>
