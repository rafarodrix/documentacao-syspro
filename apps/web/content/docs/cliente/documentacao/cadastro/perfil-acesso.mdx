# Cadastro de Perfil de Acesso

## Introdução

O cadastro de **Perfil de Acesso** é utilizado para agrupar permissões e restrições de usuários no **Syspro ERP**.

Em vez de configurar as permissões individualmente para cada usuário, a empresa pode criar perfis por função, setor ou responsabilidade, como **Vendedores**, **Financeiro**, **Compras**, **Gerência** ou **Suporte**.

Após criado, o perfil pode ser vinculado a um ou mais usuários, garantindo padronização, segurança e facilidade na gestão de acessos.

<Callout type="info" title="Objetivo do módulo">
  Centralizar permissões de acesso por perfil, facilitando o controle de módulos, telas e ações disponíveis para cada grupo de usuários.
</Callout>

## Quando utilizar

Utilize o cadastro de **Perfil de Acesso** quando for necessário controlar permissões de forma padronizada.

| Situação | Como o módulo ajuda |
|---|---|
| Criar acesso para vendedores | Permite liberar apenas módulos e ações relacionados à rotina comercial. |
| Criar acesso para financeiro | Restringe o usuário às rotinas financeiras necessárias. |
| Controlar acesso por setor | Facilita a separação de permissões entre compras, vendas, estoque, fiscal e financeiro. |
| Evitar permissões indevidas | Reduz o risco de alterações, exclusões ou acessos não autorizados. |
| Implantar novos usuários | Permite vincular rapidamente um perfil já validado ao novo usuário. |
| Auditar permissões | Facilita a revisão de acessos por grupo de usuários. |

<Callout type="success" title="Exemplo prático">
  Em uma equipe comercial, é possível criar o perfil **Vendedores** e liberar apenas as telas de venda, consulta de participantes e ações comerciais necessárias, evitando acesso indevido a módulos financeiros ou administrativos.
</Callout>

# Campos principais

## Descrição

O campo **Descrição** identifica o perfil de acesso.

Utilize nomes objetivos e relacionados à função do usuário.

Exemplos recomendados:

- `Vendedores`;
- `Financeiro`;
- `Compras`;
- `Estoque`;
- `Gerência`;
- `Suporte`;
- `Administrador`.

Evite descrições genéricas como:

- `Teste`;
- `Usuário`;
- `Perfil 1`;
- `Geral`.

<Callout type="success" title="Boa prática">
  Use nomes que representem claramente o setor ou função. Isso facilita auditoria, suporte e manutenção futura.
</Callout>

---

## Módulos

A seção **Módulos** define quais áreas do sistema estarão disponíveis para o perfil.

Exemplo de liberação:

```txt
Compra
[x] Cotação
[x] Digitação de Preço
[ ] Pedido de Compra

Comum
[x] Cadastro.Empresa
[x] Cadastro.Participante.Participante
[ ] Cadastro.Postal.Cidade
````

<Callout type="info" title="Permissão por necessidade">
  Libere apenas os módulos necessários para a função do usuário. Essa prática reduz riscos operacionais e melhora a segurança do sistema.
</Callout>

---

## Ações

A seção **Ações** define o que o usuário pode fazer dentro dos módulos liberados.

Exemplo:

```txt
[x] Alterar Cadastro AO CONSUMIDOR
[x] Alterar Limite de Crédito
[x] Alterar Preço Especial
[ ] Excluir
[x] Incluir
```

As ações podem variar conforme o módulo selecionado e a configuração da empresa.

<Callout type="warning" title="Permissão de exclusão">
  Seja criterioso ao liberar a ação **Excluir**. Uma exclusão indevida pode causar perda de dados ou exigir intervenção técnica para análise e recuperação.
</Callout>

---

## Perfil vinculado ao usuário

O perfil criado passa a ter efeito quando é vinculado ao cadastro de um usuário.

Caminho:

**Cadastro > Usuário > Usuário**

Ao editar ou incluir um usuário, selecione o perfil no campo **Perfil de Acesso**.

<Callout type="info" title="Herança de permissões">
  O usuário herda as permissões configuradas no perfil. Sempre revise o perfil antes de vinculá-lo a múltiplos usuários.
</Callout>

# Como cadastrar

<Steps>
  <Step>
    Acesse **Cadastro > Usuário > Perfil de Acesso**.
  </Step>

  <Step>
    Clique em **Incluir** para criar um novo perfil.
  </Step>

  <Step>
    No campo **Descrição**, informe um nome claro para o perfil.

```
Exemplo: **Vendedores**, **Financeiro**, **Compras** ou **Estoque**.
```

  </Step>

  <Step>
    Na área de **Módulos**, selecione as telas e menus que o perfil poderá acessar.

```
Marque apenas os módulos necessários para a rotina do grupo de usuários.
```

  </Step>

  <Step>
    Na área de **Ações**, selecione as permissões permitidas para o perfil.

```
Exemplos:

- incluir;
- alterar;
- excluir;
- alterar limite de crédito;
- alterar preço especial;
- alterar cadastro ao consumidor.
```

  </Step>

  <Step>
    Revise permissões sensíveis, principalmente ações de exclusão, alteração financeira, alteração comercial e acesso a configurações.
  </Step>

  <Step>
    Clique em **Salvar**.
  </Step>

  <Step>
    Acesse **Cadastro > Usuário > Usuário**.
  </Step>

  <Step>
    Inclua um novo usuário ou edite um usuário existente.
  </Step>

  <Step>
    No campo **Perfil de Acesso**, selecione o perfil criado.
  </Step>

  <Step>
    Clique em **Salvar**.
  </Step>

  <Step>
    Realize um teste com o usuário para validar se os módulos e ações foram liberados corretamente.
  </Step>
</Steps>

# Validação por perfil

<Tabs items={['Usuário', 'Administrador', 'Suporte']}> <Tab value="Usuário">
O usuário deve conferir:

```
- se consegue acessar os módulos necessários;
- se consegue executar as ações da sua rotina;
- se algum menu esperado não aparece;
- se alguma mensagem de acesso negado é exibida.

Caso falte acesso, o usuário deve acionar o gestor ou administrador do sistema.
```

  </Tab>

  <Tab value="Administrador">
    O administrador deve validar:

```
- se o perfil possui descrição clara;
- se os módulos liberados fazem sentido para a função;
- se ações sensíveis estão restritas;
- se a permissão de exclusão foi evitada quando não necessária;
- se o perfil foi vinculado aos usuários corretos;
- se não existem usuários com permissões excessivas.

Recomenda-se revisar perfis periodicamente.
```

  </Tab>

  <Tab value="Suporte">
    O suporte deve verificar:

```
- qual usuário está apresentando o problema;
- qual perfil está vinculado ao usuário;
- se o módulo está marcado no perfil;
- se a ação desejada está liberada;
- se o problema ocorre com um usuário ou com todos do mesmo perfil;
- se houve alteração recente no perfil.

Em caso de acesso negado, solicite print da tela, nome do usuário, perfil vinculado e ação executada.
```

  </Tab>
</Tabs>

# Boas práticas

* Crie perfis por função ou setor.
* Use descrições claras e padronizadas.
* Evite criar perfis genéricos demais.
* Não libere permissões administrativas para usuários operacionais.
* Evite liberar a ação **Excluir** sem necessidade.
* Revise permissões de crédito, desconto e preço especial.
* Teste o perfil antes de vincular a muitos usuários.
* Mantenha usuários com funções semelhantes no mesmo perfil.
* Revise perfis quando houver mudança de função do colaborador.
* Documente perfis críticos e suas permissões principais.
* Remova acessos que não são mais necessários.

<Callout type="success" title="Boa prática de segurança">
  Trabalhe com o princípio do menor privilégio: cada usuário deve ter apenas as permissões necessárias para executar sua função.
</Callout>

# Checklist antes de salvar

Antes de salvar o perfil de acesso, confira:

* descrição clara do perfil;
* setor ou função corretamente representado;
* módulos necessários selecionados;
* módulos desnecessários desmarcados;
* ações de inclusão revisadas;
* ações de alteração revisadas;
* ação de exclusão liberada apenas quando realmente necessária;
* permissões financeiras revisadas;
* permissões comerciais revisadas;
* permissões fiscais revisadas;
* permissões administrativas restritas;
* perfil testado com usuário controlado;
* usuários corretos vinculados ao perfil.

# Problemas comuns

<Accordions type="single">
  <Accordion title="O módulo não aparece para o usuário" id="modulo-nao-aparece">
    Verifique se o módulo está marcado no perfil de acesso vinculado ao usuário.

```
Também confirme se o usuário está com o perfil correto e se o cadastro foi salvo após a alteração.
```

  </Accordion>

  <Accordion title="O usuário acessa a tela, mas não consegue incluir" id="nao-consegue-incluir">
    Verifique se a ação **Incluir** está liberada no perfil.

```
O acesso ao módulo permite abrir a tela, mas a ação precisa estar liberada separadamente quando o sistema utiliza permissões por ação.
```

  </Accordion>

  <Accordion title="O usuário não consegue alterar um cadastro" id="nao-consegue-alterar">
    Confira se a ação **Alterar** está marcada no perfil.

```
Também verifique se existe alguma regra específica para o módulo ou cadastro em questão.
```

  </Accordion>

  <Accordion title="O usuário não consegue excluir registros" id="nao-consegue-excluir">
    Verifique se a ação **Excluir** está liberada.

```
Caso não esteja, confirme com o administrador se essa permissão deve realmente ser concedida.
```

  </Accordion>

  <Accordion title="O usuário está com acesso maior do que deveria" id="acesso-maior-que-deveria">
    Revise o perfil vinculado ao usuário.

```
Verifique se ele foi associado ao perfil correto e se o perfil possui módulos ou ações liberadas indevidamente.
```

  </Accordion>

  <Accordion title="Alterei o perfil, mas o usuário continua sem acesso" id="alteracao-nao-aplicada">
    Confirme se o perfil foi salvo corretamente.

```
Quando aplicável, peça para o usuário sair e entrar novamente no sistema para recarregar as permissões.
```

  </Accordion>

  <Accordion title="Não sei qual permissão liberar" id="qual-permissao-liberar">
    Identifique a rotina que o usuário precisa executar, o módulo acessado e a ação desejada.

```
Em caso de dúvida, libere inicialmente apenas o necessário e teste com o usuário.
```

  </Accordion>
</Accordions>

# Resumo

O cadastro de **Perfil de Acesso** permite organizar permissões por função, setor ou responsabilidade dentro do Syspro ERP.

Com ele, a empresa reduz retrabalho, melhora a segurança e facilita a manutenção de usuários, evitando permissões excessivas ou acessos indevidos.

  Crie perfis claros, libere apenas os módulos necessários, restrinja ações sensíveis e teste as permissões antes de aplicar o perfil a vários usuários.
</Callout>
