## Plataforma Remota — backlog

**Novos recursos**

**Mapa de cobertura remota** — grid ou mapa de regiões do Brasil com densidade de hosts por UF. Útil para gerenciar cobertura de clientes.

**Timeline de heartbeats por host** — gráfico de linha mostrando saúde do agente ao longo do tempo (latência, falhas, reinicializações). Hoje só existe `lastHeartbeatAt` sem histórico.

**Dashboard de status em tempo real** — SSE ou polling curto mostrando quais hosts estão `ACTIVE`, `INACTIVE`, `MAINTENANCE` com atualização automática sem reload.

**Sessões remotas inline** — na página de detalhes do host, exibir o histórico completo de sessões com duração, operador, ticket vinculado e possibilidade de reabrir uma sessão expirada.

**Bulk actions na listagem** — selecionar múltiplos hosts e aplicar: trocar empresa, mudar status, baixar instaladores em lote (zip).

**Notificação proativa de host offline** — quando um host ativo não envia heartbeat por mais de N minutos, acionar notificação no painel e opcionalmente webhook.

**Versionamento de agente** — detectar quando `agentVersion` está desatualizado em relação à versão configurada no módulo e sinalizar no card do host.

**Notas ricas por host** — hoje `notes` é texto plano. Suporte a markdown com preview, histórico de edições (quem editou e quando).

**Auditoria de sessões** — endpoint e tela listando todas as sessões de um período, com filtros por operador, empresa, duração, ticket.

**Portal do cliente para Plataforma Remota** — `CLIENTE_ADMIN` já tem acesso mas a interface é a mesma do admin. Criar visão simplificada: lista dos próprios hosts, status, botão de solicitar sessão sem expor configurações internas.

```typescript
    // Unifica empresa e tags em uma lista plana para o RustDesk
    tags: [
      `Empresa: ${dev.company.name}`,
      ...(dev.company.cnpj ? [`CNPJ: ${dev.company.cnpj}`] : []),
      ...dev.tags.map(t => t.name)
    ],
    hash: dev.updatedAt.getTime().toString(),
  }));

  return NextResponse.json(addressBook);
}
```

### ?? Dicas de Manutenção e Performance

1.  **Cache do Address Book**: O RustDesk consulta a API com frequência. Use `revalidatePath` ou `cache` do Next.js para não sobrecarregar o banco de dados.
2.  **Hash do Dispositivo**: O campo `hash` no JSON do Address Book é obrigatório. Sempre que mudar uma tag ou o nome no seu portal, atualize o `updatedAt` para que o RustDesk perceba a mudança.
3.  **Segurança**: Use o `middleware.ts` do Next.js para proteger essas rotas, exigindo o Bearer Token que você emitiu no login.

Com essa estrutura, você elimina a dependência de repositórios externos e mantém o controle total sobre como seus clientes são organizados e acessados.
