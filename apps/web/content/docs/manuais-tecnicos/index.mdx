## Estrutura do projeto (monorepo)

- `apps/web`: frontend Next.js (portal + docs + rotas API do app web)
- `apps/api`: shell HTTP dedicado para backend desacoplado
- `apps/mobile`: shell mobile (runtime em evolucao)
- `packages/database`: schema Prisma, migrations e seed
- `packages/api`: contratos e routers compartilhados
- `packages/remote-domain`: casos de uso e contratos do dominio remoto
- `scripts/`: utilitarios operacionais (check, build e automacoes de suporte)

## Comandos operacionais (referencia rapida)

### Workspace (raiz)

```bash
npm run dev
npm run build
npm run lint
npm run typecheck
npm run test
```

### Banco e Prisma

```bash
npm run db:validate
npm run db:generate
npm run db:migrate   # dev
npm run db:deploy    # producao
npm run db:seed:remote
```

### Web (portal + docs)

```bash
npm run dev -w @dosc-syspro/web
npm run build -w @dosc-syspro/web
npm run lint -w @dosc-syspro/web
npm run typecheck -w @dosc-syspro/web
npm run docs:check -w @dosc-syspro/web
```

### API e mobile (shells dedicados)

```bash
npm run dev:api
npm run build:api
npm run typecheck:api

npm run dev:mobile
npm run build:mobile
npm run typecheck:mobile
```

### Packages compartilhados (dominio e router)

```bash
npm run typecheck -w @dosc-syspro/api
npm run test -w @dosc-syspro/api
npm run typecheck -w @dosc-syspro/remote-domain
```

### Paridade com CI (checagem final antes do PR)

```bash
npm run typecheck:contracts
npm run typecheck
npm run typecheck:api
npm run typecheck -w @dosc-syspro/api
npm run test -w @dosc-syspro/api
npm run build
npm run build:api
npm run build:mobile
```

### E2E (Playwright)

```bash
npm run test:e2e:install
npm run test:e2e
npm run test:e2e:headed
```

## Uso recomendado deste indice

1. Comece por arquitetura (`arquitetura-aplicacao-monorepo` + `estrutura-projeto`).
2. Para remoto, siga `arquitetura-remote` e depois backlog.
3. Para entrega, use os comandos desta pagina como baseline de execucao.
