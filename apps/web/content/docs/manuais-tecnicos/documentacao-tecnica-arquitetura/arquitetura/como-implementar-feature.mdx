## Exemplo Prático: "Gerenciamento de Auditorias"

Vamos implementar uma feature de auditoria de ações de usuários (listar, filtrar, exportar).

---

## Fase 1: Planejamento e Contratos (30 min)

### Step 1: Definir Scope e Decisões

Responda estas perguntas **antes** de escrever código:

- **O que a feature faz?**
  - Listar ações auditadas por usuário e período
  - Filtrar por tipo de ação, resultado, IP
  - Exportar como CSV

- **Quem pode acessar?**
  - ADMIN: ver todas ações
  - SUPORTE: ver ações de sua empresa
  - USER: ver apenas suas próprias ações

- **Dados necessários?**
  - Preciso de tabela nova? Sim: `audit_log`
  - Integrações externas? Não
  - Cache? Sim, resultados de filtro (5 min TTL)

- **Onde vive?**
  - Feature: `src/features/audit`
  - API: `packages/api/src/routers/audit.ts`
  - Contrato: `packages/contracts/src/audit.ts`

### Step 2: Criar Contrato em Shared

```typescript
// packages/contracts/src/audit.ts
import { z } from "zod";

export const auditActionTypeSchema = z.enum([
    "USER_LOGIN",
    "USER_LOGOUT",
    "USER_CREATE",
    "COMPANY_UPDATE",
    "TICKET_CREATED"
]);

export const auditLogSchema = z.object({
    id: z.string(),
    userId: z.string(),
    action: auditActionTypeSchema,
    resource: z.string().optional(),
    resourceId: z.string().optional(),
    result: z.enum(["SUCCESS", "FAILED"]),
    ip: z.string().nullable(),
    timestamp: z.date(),
    details: z.record(z.unknown()).optional()
});

export type AuditLog = z.infer<typeof auditLogSchema>;
export type AuditActionType = z.infer<typeof auditActionTypeSchema>;

export const listAuditLogsInputSchema = z.object({
    userId: z.string().optional(),
    action: auditActionTypeSchema.optional(),
    result: z.enum(["SUCCESS", "FAILED"]).optional(),
    from: z.date().optional(),
    to: z.date().optional(),
    limit: z.number().int().min(1).max(100).default(50),
    offset: z.number().int().min(0).default(0)
});

export type ListAuditLogsInput = z.infer<typeof listAuditLogsInputSchema>;
export type ListAuditLogsOutput = {
    items: AuditLog[];
    total: number;
    hasMore: boolean;
};
```

### Step 3: Criar Schema Prisma

```prisma
// prisma/schema.prisma
model AuditLog {
    id String @id @default(cuid())
    
    // Autor da ação
    userId String
    user User @relation(fields: [userId], references: [id], onDelete: Cascade)
    
    // O quê aconteceu
    action String  // USER_LOGIN, COMPANY_UPDATE, etc
    resource String? // "user", "company", "ticket"
    resourceId String?
    
    // Resultado
    result String // "SUCCESS", "FAILED"
    
    // Context
    ip String?
    userAgent String?
    timestamp DateTime @default(now())
    
    // Metadata flexível
    details Json?
    
    // Tenant
    companyId String
    company Company @relation(fields: [companyId], references: [id], onDelete: Cascade)
    
    // Índices para query frequente
    @@index([userId, timestamp])
    @@index([companyId, timestamp])
    @@index([action, timestamp])
    @@index([result])
}
```

Depois rodar:
```bash
npx prisma migrate dev --name add_audit_log
```

## Fase 3: Application Layer (1 hora)

### Step 7: Implementar Queries (Leitura)

```typescript
// src/features/audit/application/queries.ts
"use server";

import { getProtectedSession } from "@/lib/auth-helpers";
import { auditReadRepository } from "@/features/audit/infrastructure/repositories/audit-read.repository";
import type { AuditDashboardData, AuditItemSummary } from "@/features/audit/domain/model";
import { Role } from "@prisma/client";

// Query 1: Dashboard (últimas 10 ações)
export async function getAuditDashboardData(): Promise<AuditDashboardData> {
    const session = await getProtectedSession();
    
    // Validar acesso (ADMIN ou SUPORTE)
    if (!["ADMIN", "SUPORTE"].includes(session.user.role)) {
        throw new Error("FORBIDDEN");
    }
    
    // Definir scope based on role
    const userId = session.user.role === "ADMIN" ? undefined : session.user.id;
    const companyId = session.user.role === "ADMIN" ? undefined : session.user.companyId;
    
    // Buscar dados
    const recentLogs = await auditReadRepository.findByUserId(userId, { limit: 10 });
    const failureRate = await auditReadRepository.getFailureRate(companyId, 1); // Últimas 24h
    
    // Mapear para view model
    const recentActions: AuditItemSummary[] = recentLogs.map(log => ({
        id: log.id,
        userName: log.user?.name ?? "Unknown",
        action: log.action,
        resource: log.resource,
        result: log.result as "SUCCESS" | "FAILED",
        timestamp: log.timestamp,
        relativeTime: formatRelativeTime(log.timestamp)
    }));
    
    return {
        recentActions,
        statistics: {
            totalToday: recentLogs.length,
            failureRatePercent: Math.round(failureRate * 100),
            uniqueUsersToday: new Set(recentLogs.map(l => l.userId)).size
        }
    };
}

// Query 2: Lista filtrada
export async function listAuditLogsQuery(input: ListAuditLogsInput): Promise<AuditLog[]> {
    const session = await getProtectedSession();
    
    // Applica scope automático baseado em role
    const enrichedInput = {
        ...input,
        ...(session.user.role === "USER" && { userId: session.user.id }),
        ...(session.user.role === "SUPORTE" && { companyId: session.user.companyId })
    };
    
    const { items } = await auditReadRepository.listFiltered(enrichedInput);
    return items;
}
```

## Fase 4: Infrastructure Layer (45 min)

### Step 9: Implementar Repositories

```typescript
// src/features/audit/infrastructure/repositories/audit-read.repository.ts
import { prisma } from "@/lib/prisma";
import { redis } from "@/lib/redis";
import type { AuditReadRepository } from "@/features/audit/domain/repositories/audit-read.repository";

export const auditReadRepository: AuditReadRepository = {
    
    async findByUserId(userId, options = {}) {
        const cacheKey = `audit:user:${userId}`;
        
        // Tentar cache
        const cached = await redis.get(cacheKey);
        if (cached) return JSON.parse(cached);
        
        // Fetch do banco
        const logs = await prisma.auditLog.findMany({
            where: { userId },
            orderBy: { timestamp: "desc" },
            take: options.limit ?? 50,
            include: { user: true }
        });
        
        // Cache 5 minutos
        await redis.setex(cacheKey, 300, JSON.stringify(logs));
        
        return logs;
    },
    
    async listFiltered(input) {
        const query = prisma.auditLog.findMany({
            where: {
                ...(input.userId && { userId: input.userId }),
                ...(input.action && { action: input.action }),
                ...(input.result && { result: input.result }),
                ...(input.from && { timestamp: { gte: input.from } }),
                ...(input.to && { timestamp: { lte: input.to } })
            },
            orderBy: { timestamp: "desc" },
            take: input.limit,
            skip: input.offset,
            include: { user: true }
        });
        
        const total = await prisma.auditLog.count({
            where: query.where
        });
        
        const items = await query;
        
        return { items, total };
    },
    
    async getFailureRate(companyId, days = 1) {
        const since = new Date(Date.now() - days * 24 * 60 * 60 * 1000);
        
        const total = await prisma.auditLog.count({
            where: { companyId, timestamp: { gte: since } }
        });
        
        const failures = await prisma.auditLog.count({
            where: { companyId, result: "FAILED", timestamp: { gte: since } }
        });
        
        return total === 0 ? 0 : failures / total;
    },
    
    async getActionTypesForUser(userId) {
        const results = await prisma.auditLog.groupBy({
            by: ["action"],
            where: { userId }
        });
        
        return results.map(r => r.action);
    }
};
```

### Step 10: Criar Gateway de Logging

```typescript
// src/features/audit/infrastructure/gateways/audit-logger.gateway.ts
import { recordAuditLogAction } from "@/features/audit/application/actions";

// Usar em qualquer lugar do app
export async function auditLog(
    action: string,
    resource?: string,
    resourceId?: string,
    result: "SUCCESS" | "FAILED" = "SUCCESS",
    details?: Record<string, unknown>
) {
    try {
        await recordAuditLogAction(action, resource, resourceId, result, details);
    } catch (error) {
        // Fallback: log para stderr, não quebra a ação
        console.error("[AuditLog Fallback]:", error);
    }
}
```

```typescript
// src/features/audit/interface/components/audit-list.tsx
"use client";

import { useTransition } from "react";
import { listAuditLogsQuery } from "@/features/audit/application/queries";
import type { AuditLog } from "@dosc-syspro/contracts";

export function AuditList() {
    const [logs, setLogs] = useState<AuditLog[]>([]);
    const [pending, startTransition] = useTransition();
    
    async function handleFilter(filters: ListAuditLogsInput) {
        startTransition(async () => {
            const result = await listAuditLogsQuery(filters);
            setLogs(result);
        });
    }
    
    return (
        <div>
            <AuditFilters onFilter={handleFilter} />
            <AuditTable logs={logs} loading={pending} />
        </div>
    );
}
```

## Fase 6: API (tRPC Router) (30 min)

### Step 14: Criar tRPC Router

```typescript
// packages/api/src/routers/audit.ts
import { createRouter, defineQuery, defineMutation } from "../router";
import { listAuditLogsInputSchema } from "@dosc-syspro/contracts";

export const auditRouter = createRouter({
    
    list: defineQuery<{ input: ListAuditLogsInput }, AuditLog[]>({
        auth: "authenticated",
        handler: async ({ ctx, input }) => {
            const logs = await auditReadRepository.listFiltered({
                ...input,
                ...(ctx.session.user.role === "USER" && { userId: ctx.session.user.id })
            });
            
            ctx.logger.info("audit.list.queried", {
                userId: ctx.session.user.id,
                count: logs.items.length
            });
            
            return logs.items;
        }
    }),
    
    dashboard: defineQuery<void, AuditDashboardData>({
        auth: "authenticated",
        handler: async ({ ctx }) => {
            if (!["ADMIN", "SUPORTE"].includes(ctx.session.user.role)) {
                throw new ApiError("FORBIDDEN", "Admin access required");
            }
            
            return await getAuditDashboardData();
        }
    })
});
```

## Próximas Features

Use exatamente o mesmo padrão:

1. **Contratos** em `packages/contracts`
2. **Domain** + **Application** em `src/features/<feature>`
3. **Infrastructure** (repositories, gateways) in `infrastructure/`
4. **Interface** (components, page) em `interface/`
5. **API** em `packages/api`

Consistência = escalabilidade! 🎯
