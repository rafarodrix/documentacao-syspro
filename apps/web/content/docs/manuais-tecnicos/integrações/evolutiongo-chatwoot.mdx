# 📡 Integração Evolution Go + Chatwoot (Orquestrada via Backend)

## Introdução

A integração entre **Evolution Go** e **Chatwoot** permite transformar o WhatsApp em um canal omnichannel dentro do seu portal de suporte.

Nesta arquitetura, utilizamos o **Backend central do Syspro (`apps/api` em NestJS)** como orquestrador. Isso nos permite cruzar os dados do WhatsApp com o nosso banco de dados (Prisma), enriquecendo os contatos automaticamente (identificando a Empresa do cliente) antes de enviá-los para o painel de atendimento.

---

### Fluxo principal

```text
WhatsApp → Evolution Go → (Webhook) → Backend NestJS → (API) → Chatwoot
Chatwoot → (Webhook) → Backend NestJS → (API) → Evolution Go → WhatsApp
```

---

## 🏗️ Papel de cada componente

### Evolution Go

* Gateway WhatsApp.
* Recebe/envia mensagens.
* Gerencia internamente os webhooks de sincronização com o Chatwoot.
* Cria contatos e conversas automaticamente na Caixa de Entrada (Inbox) configurada.

### Chatwoot

* Interface de atendimento omnichannel.
* Gestão de conversas e distribuição para agentes humanos.
* Dispara eventos de volta para o Evolution Go quando um agente responde.

---

### Motivos para a Mudança (Nova Rota):

* **Redução de Complexidade:** Eliminação do orquestrador NestJS.
* **Menor Latência:** As mensagens fluem diretamente entre os serviços.
* **Manutenção Simplificada:** Menos código customizado e uso de funcionalidades nativas já suportadas pela comunidade do Evolution API.

# 🔌 Implementação Técnica

## 1. Configuração no Evolution Go

A integração é configurada diretamente na instância do Evolution Go, ativando a feature do Chatwoot.

### Endpoint para ativar a integração (Evolution API)

```bash
POST /chatwoot/set/{instance}
```

### Payload de Configuração

```json
{
  "enabled": true,
  "accountId": "2",
  "token": "seu_token_de_acesso_ao_chatwoot",
  "url": "http://chat.trilinksoftware.com.br:3000",
  "signMsg": true,
  "reopenConversation": true,
  "conversationPending": false
}
```

* **accountId:** O ID da conta no Chatwoot.
* **token:** O *Access Token* de um usuário administrador no Chatwoot.
* **url:** A URL base da instalação do Chatwoot.
* **signMsg:** Adiciona a assinatura (nome) do agente na mensagem enviada ao WhatsApp.

---

## 2. Processamento Interno (Evolution)

Ao ativar essa integração, o Evolution Go se encarrega de:

1. Criar uma nova **Caixa de Entrada (Inbox)** no Chatwoot para a instância do WhatsApp (se não existir).
2. Criar ou sincronizar o **Contato** no Chatwoot sempre que uma mensagem for recebida.
3. Criar e gerenciar o fluxo da **Conversa**, repassando todas as mensagens recebidas via WhatsApp diretamente para o painel do Chatwoot.

---

## 3. Fluxo reverso (Chatwoot → WhatsApp)

O Chatwoot, através do seu próprio sistema de Webhooks configurado automaticamente pelo Evolution na Inbox, avisa o Evolution sobre novas mensagens enviadas pelos atendentes.

### Evento capturado pelo Evolution
* `message_created` (Quando um atendente envia uma resposta).

O Evolution captura esse evento e realiza o disparo da mensagem (texto, imagem, áudio, etc.) para o número de WhatsApp correspondente.

# 🌐 Variáveis de ambiente importantes

Apesar da integração ocorrer via requisições à API, certifique-se de que os seguintes acessos estão disponíveis:

```env
# Evolution Go
EVOLUTION_API_URL=https://api.trilinksoftware.com.br
EVOLUTION_API_KEY=sua_chave_global_aqui

# Chatwoot
CHATWOOT_URL=http://chat.trilinksoftware.com.br:3000
CHATWOOT_ACCOUNT_ID=2
CHATWOOT_API_TOKEN=seu_token_admin_aqui
```

---

# 🔐 Segurança

* Utilize chaves de API fortes (`GLOBAL_API_KEY` e `CHATWOOT_API_TOKEN`).
* Garanta que a comunicação entre o Evolution Go e o Chatwoot ocorra em uma rede segura (preferencialmente com HTTPS/TLS nas URLs).
* Rotacione as credenciais periodicamente.

---

# 🚀 Benefícios desta arquitetura

### Performance e Estabilidade

* Eliminação de pontos únicos de falha customizados (o backend intermediário).
* Sincronização em tempo real otimizada pelos criadores do Evolution API.

### Facilidade de Configuração

* Configuração feita em um único request REST para o Evolution.
* A criação de inboxes, webhooks e contatos ocorre magicamente de forma transparente.
