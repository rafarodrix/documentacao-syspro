Esta pagina define o padrao oficial para criar e manter paginas da documentacao.
Escopo: componentes disponiveis, estrutura de frontmatter, padroes de escrita e checklist antes de publicar.

## Quem pode acessar

Esta pagina e exclusiva para `ADMIN`.

## Estrutura minima de uma pagina MDX

Use este frontmatter como base:

```yaml
---
title: Titulo da pagina
description: Resumo curto e objetivo da pagina.
lastUpdated: 2026-03-22
owner: Equipe de Documentacao
status: published # draft | review | published | archived
featureStatus: beta # opcional: new | deprecated | beta | experimental
sinceVersion: v2.4 # opcional
tags:
  - docs
  - operacao
---
```

## Componentes disponiveis no MDX

### Admonitions

```mdx
<Tip>Boa pratica para acelerar o processo.</Tip>
<Note>Informacao complementar importante.</Note>
<Warning>Risco operacional se feito fora de ordem.</Warning>
<Danger>Acao critica ou irreversivel.</Danger>
```

### Tabs de codigo (preferencia global persistida)

~~~mdx
<CodeTabs items={['TypeScript', 'JavaScript']}>
  <CodeTab value="TypeScript">
    ```ts
    const user = { id: '1', role: 'ADMIN' };
    ```
  </CodeTab>
  <CodeTab value="JavaScript">
    ```js
    const user = { id: '1', role: 'ADMIN' };
    ```
  </CodeTab>
</CodeTabs>
~~~

### Playground inline (editavel e executavel)

```mdx
<PlaygroundInline
  title="Teste rapido"
  code={`const total = [10, 20, 30].reduce((acc, n) => acc + n, 0);
console.log('Total:', total);`}
/>
```

## Padrao de escrita

1. Use titulo claro com foco em tarefa.
2. Abra com "Objetivo" e, quando aplicavel, "Pre-requisitos".
3. Use secoes curtas com passos numerados.
4. Prefira exemplos praticos e validaveis.
5. Finalize com "Erros comuns" ou "Checklist final" quando fizer sentido.

## Padrao de arquivos e slugs

1. Nome de arquivo em `kebab-case`.
2. Sem espacos e sem caracteres especiais.
3. Slug estavel para evitar quebrar links.
4. Ao renomear pagina, cadastrar redirect no projeto.

## Checklist antes de publicar

- Frontmatter completo e valido.
- Links internos testados.
- Titulo e descricao coerentes com o conteudo.
- Conteudo revisado por outro membro (quando status = review).
- `docs:check` passando no CI.
