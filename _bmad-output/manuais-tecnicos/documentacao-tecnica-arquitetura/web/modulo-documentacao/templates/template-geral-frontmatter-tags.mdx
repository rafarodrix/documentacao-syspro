## Objetivo

Padronizar como preencher metadados dos arquivos MDX para melhorar busca, governanca e manutencao.

## Frontmatter base (obrigatorio)

```yaml
---
title: [titulo da pagina]
description: [resumo curto e direto da pagina]
lastUpdated: YYYY-MM-DD
owner: [responsavel pelo conteudo]
status: draft | review | published | archived
tags:
  - [tag-1]
  - [tag-2]
audience:
  - [perfil-1]
  - [perfil-2]
---
```

## Funcao de cada campo

| Campo | Funcao | Regra pratica |
| --- | --- | --- |
| `title` | Nome da pagina | Claro e orientado ao tema |
| `description` | Resumo para contexto e SEO interno | 1 frase objetiva |
| `lastUpdated` | Data da ultima revisao relevante | Formato `YYYY-MM-DD` |
| `owner` | Responsavel por manutencao do conteudo | Time ou pessoa |
| `status` | Estado editorial | Use `published` em pagina ativa |
| `tags` | Classificacao para busca e relacoes | 2-4 tags relevantes |
| `audience` | Publico-alvo do conteudo | Perfis que usam a pagina |

## Opcoes de status

- `draft`: em elaboracao, nao pronto para consumo amplo.
- `review`: pronto para revisao tecnica/funcional.
- `published`: conteudo oficial publicado.
- `archived`: historico, nao recomendado para novo uso.

## Opcoes de audience

- `cliente_admin`: gestor/administrador do cliente.
- `cliente_user`: usuario operacional do cliente.
- `suporte`: equipe de suporte.
- `admin`: equipe administrativa interna.
- `developer`: equipe tecnica/desenvolvimento.

## Catalogo de tags recomendadas

### Por trilha principal

- `manual`
- `duvidas`
- `treinamento`
- `suporte`
- `manuais-tecnicos`

### Por tipo de conteudo

- `template`
- `guia`
- `procedimento`
- `playbook`
- `arquitetura`
- `integracao`
- `operacao`
- `checklist`

### Por dominio/modulo (exemplos)

- `cadastro`
- `estoque`
- `compra`
- `financeiro`
- `fiscal`
- `vendas`
- `relatorios`
- `caixa`
- `dre`
- `boleto`

## Padrão recomendado para tags

1. Sempre incluir a trilha principal (`manual`, `duvidas`, etc.).
2. Incluir 1 tag de modulo/submodulo.
3. Incluir 1 tag especifica da pagina (slug curto).
4. Evitar excesso de tags (maximo recomendado: 4).

## Exemplos completos

### Exemplo 1: Manual

```yaml
---

status: published

  - manual

  - participante
audience:
  - cliente_admin
  - cliente_user
  - suporte

```
